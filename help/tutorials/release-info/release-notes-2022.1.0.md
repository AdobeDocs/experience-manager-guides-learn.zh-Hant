# 1月發行 [!DNL Adobe Experience Manager Guides] as a Cloud Service

## 升級至1月版

升級您目前的 [!DNL Adobe Experience Manager Guides] as a Cloud Service(稍後稱為 [!DNL AEM Guides] as a Cloud Service)，請執行下列步驟：
1. 查看Cloud Services的Git程式碼，並切換至Cloud Services管道中所設定的分支，該管道與您要升級的環境對應。
1. 更新 `<dox.version>` 屬性 `/dox/dox.installer/pom.xml` 檔案的Git程式碼。
1. 提交變更並執行Cloud Services管道，以升級至 [!DNL AEM Guides] as a Cloud Service。

## 新增功能和功能改善

### 文章發佈

在1月版本中，我們導入了整合在網頁編輯器中的文章發佈功能。 您可以使用文章發佈功能逐步產生一或多個主題的輸出，或將內容發佈到知識庫平台。

此功能可讓使用者以加法方式建立DITA地圖，並在準備好時發佈主題。 發佈地圖後，請使用文章發佈功能，只對更新後的文章實現增量發佈。

![文章發佈](assets/article-based-publishing.png)

除了AEM之外，您還可以使用此獨特功能將文章發佈到任何知識庫門戶，如Salesforce。 此功能也隨附OOTB內容範本，此範本以AEM核心元件為基礎，可讓您建立技術內容的知識型存放庫。 此模板的優點在於它完全可定製，以符合您的組織要求，並且還可支援企業內部網路門戶等使用案例。
您也可以根據文章的檔案狀態和修改時間來篩選文章。

這項隨選隨需文章發佈不僅可讓您完全控制內容發佈，而且可縮短發布更新內容的整體時間。
使用此範本發佈文章時，也可以將中繼資料傳遞至您發佈的頁面。
如需詳細資訊，請參閱 *從網頁編輯器發佈以文章為基礎* 中。

### 改良的網頁編輯器

網頁編輯器中推出了許多增強功能和新功能：

* 網頁編輯器中也新增了對主旨方案的支援。 您現在可以使用「主旨配置」面板來建立和使用主旨配置。 除了新增主題配置外，您現在可以使用自己的公司中繼資料和分類法。

![主體方案](assets/subject-scheme-panel.png)

* 此版本引入了新的辭匯表熱點工具，以批量管理辭匯表。 使用此工具，您可以快速將文字轉換為字彙表和字彙表，並針對所選地圖或開啟的主題大量轉換為詞語。

![辭匯熱點](assets/glossary-hotspot-tool.png)

* 在「可重複使用的內容」面板中新增重新整理功能，可讓您快速重新整理參考檔案中可重複使用的內容。
* 新的工作副本指示器顯示檔案的當前（工作副本）是否與保存的版本同步。

![版本指標](assets/version-update-indicator.png)

* 增強「儲存庫面板」和「檔案瀏覽」對話方塊中的搜尋篩選，提供更多篩選選項，供您進一步自訂。

![在儲存庫中搜尋篩選器](assets/repository-filter-search.png)

* 您現在可以從Web編輯器上傳.docx檔案。

### 使用FrameMaker製作

現在，您可以在FrameMaker中撰寫和發佈檔案。 FrameMaker隨附現成可用的Adobe Experience Manager連接器。 在FrameMaker中，您獲得了一個易於使用的介面，該介面允許您在分佈式協作環境中維護文檔的版本。

建立內容後，FrameMaker允許您以不同格式發佈文檔 — PDF、HTML5、EPUB和DITA。 您也可以執行各種檔案管理操作，例如結帳、含相依項的結帳、簽入、重新整理等。
使用FrameMaker製作 [!DNL AEM Guides] as a Cloud Service使用FrameMaker 2020.4版及更高版本。

### 新的翻譯控制面板

Web編輯器中引入了新的翻譯控制面板，其功能如下：

* 排序、搜尋和篩選主題清單。
* 依參考類型篩選內容 — 直接或間接參考。
* 啟動翻譯請求時，可輕鬆導覽以尋找現有專案。
* 引入多語言翻譯機制，以避免在為多種語言起始翻譯請求時為每種語言建立多個專案。
* 導入了在映射儀表板中隱藏翻譯頁簽的配置。 依預設，它會顯示。 您可以選擇使用地圖控制面板或網頁編輯器來翻譯內容。

![翻譯控制面板](assets/translation-from-web-editor.png)

### 增強的發佈功能

* 作者現在可以將地圖和主題層級中繼資料傳遞至DITA-OT發佈。 設計自訂PDF範本時，如果要使用檔案中繼資料屬性（例如標籤、作者、檔案狀態等），這個功能會很好用。

![DITA-OT中繼資料](assets/custom-meta-data-output-preset.png)

* 已新增新設定，讓使用者可保留或刪除當 **刪除和建立** 選項用於AEM網站輸出產生。

### 改善檔案處理

在AEM Assets中處理檔案時，現在可以看到下列改善項目：
* 推出新的檔案上傳體驗，以及選擇衝突解決策略的新對話方塊。

![檔案上傳衝突](assets/file-upload-name-conflict.png)

* 能夠建立新版本的已上傳檔案，並防止覆寫已簽出的檔案。
* 現在，您可以直接從「版本記錄」檢視中查看影像的預覽。 此外，對於DITA和非DITA檔案，版本歷史記錄會分別顯示當前版本資訊。

![版本歷史記錄縮圖](assets/version-history-preview-image.png)

* 每當使用者建立DITA檔案時，預設檔案名稱會以小寫顯示，與原生AEM資料夾建立案例內嵌。

### 新的報表匯出功能

報表在識別內容的健全狀態時非常有用。 [!DNL AEM Guides] as a Cloud Service提供各種報表，以控制您的內容。 現在，您不僅可以檢視報表，還可以將報表資料匯出為CSV檔案，以便檢視並與您的大型團隊共用。 報表資料可讓您快速查看任何損壞的連結或遺失的影像。

![報表匯出](assets/export-report.png)

### 改善Oxon DAM更新體驗

當您從氧氣中的AEM伺服器重新整理檔案時，如果您目前的氧氣工作階段中有未儲存的檔案，則會顯示警告訊息。 您可以選擇取消刷新操作以保存任何未保存的檔案。 若沒有此功能，使用者的檔案中會遺失任何未儲存的資訊。


### 其他功能增強功能

* 您現在可以建立新 **Dita專案** 範本 **/apps/projects/templates** 路徑。
* 現在下載預設值 **ui_config.json** 檔案。 這可用來合併現有 **ui_config.json** 檔案。
* 即使有新版本的JS檔案，您也不需要清除瀏覽器快取。

## 已知問題

Adobe已識別 [!DNL AEM Guides] as a Cloud Service於2022年1月發行。


### 因應措施的已知問題

請針對下列已知問題使用指定的因應措施：

* Mac上的Oxon連接器無法使用Web驗證。
   **因應措施**:現在使用Windows上的氧氣連接器。

* 在Firefox瀏覽器中，若不開啟並排檢視，就無法匯入檢閱註解。
   **因應措施**:現在請使用Chrome瀏覽器。

* 在移動檔案名稱中有空格的影像或多媒體檔案時，引用中斷。
   **因應措施**:更名檔案，並在移動前從檔案名中刪除空格。

* 最新版Chrome瀏覽器中的對應控制面板不會間歇性載入。
   **因應措施**:重新整理對應控制面板頁面。

### 其他已知問題

* 如果氧氣連接了 [!DNL AEM Guides] 使用web驗證的解決方案，然後註銷失敗。
* 無法將審核任務重新分配給用戶。
* 地圖收集UI中出現問題，例如文字扭曲和 **全選** 功能無法正常運作。
