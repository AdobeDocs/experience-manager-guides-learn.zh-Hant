# 1月份 [!DNL Adobe Experience Manager Guides] as a Cloud Service

## 升級到1月份版本

升級當前 [!DNL Adobe Experience Manager Guides] as a Cloud Service(後者稱為 [!DNL AEM Guides] as a Cloud Service)，執行以下步驟：

## 新增功能和功能改善

### 基於文章的出版

在1月份發佈後，我們引入了一個整合在Web編輯器中的基於文章的發佈功能。 您可以使用基於文章的發佈功能以增量方式生成一個或多個主題的輸出，或將內容發佈到知識庫平台。

此功能允許用戶以添加方式構建DITA映射，並在主題準備好時發佈主題。 發佈地圖後，使用基於文章的發佈功能僅實現更新文章的增量發佈。

![基於文章的出版](assets/article-based-publishing.png)

除此之外AEM，您還可以使用此獨特功能將文章發佈到任何知識庫門戶（如Salesforce）。 此功能還附帶了一個OOTB內容模板，該模板構建在核心元件AEM之上，使您能夠建立基於知識的技術內容儲存庫。 此模板的最好之處是，它完全可定製，以滿足您的組織需求，還可支援企業內部網門戶等使用案例。
您還可以根據文章的文檔狀態和修改時間來篩選文章。

此基於移動需求的文章發佈不僅讓您能夠完全控制內容發佈，而且還減少了發佈更新內容的總體時間。
使用此模板發佈文章時，它還可以將元資料傳遞給已發佈的頁面。
有關詳細資訊，請參閱 *基於文章的Web編輯器發佈* 的上界。

### 改進的Web編輯器

Web編輯器中引入了許多增強功能和新功能：

* 在Web編輯器中也添加了對主題方案的支援。 現在，您可以使用「主題方案」面板建立和使用主題方案。 通過添加主題方案，您現在可以使用自己的公司元資料和分類。

![主題方案](assets/subject-scheme-panel.png)

* 在此版本中引入了一個新的辭彙熱點工具，以批量管理辭彙表。 使用此工具，您可以快速將文本轉換為辭彙表，並將辭彙表轉換為選定地圖或開啟主題的大量術語。

![辭彙熱點](assets/glossary-hotspot-tool.png)

* 在「可重用內容」面板中添加了刷新功能，允許您快速刷新引用檔案中可重用的內容。
* 新的工作副本指示器顯示檔案的當前（工作副本）是否與保存的版本同步。

![版本指示器](assets/version-update-indicator.png)

* 「儲存庫面板」和「檔案瀏覽」對話框中的搜索篩選器已增強，可提供更多篩選選項，這些選項可進一步自定義。

![在儲存庫中搜索篩選器](assets/repository-filter-search.png)

* 您現在可以從Web編輯器上載.docx檔案。

### 使用FrameMaker的作者

現在，您可以在FrameMaker中建立和發佈文檔。 FrameMaker附帶一個到Adobe Experience Manager的現成連接器。 在FrameMaker中，您可以獲得一個易於使用的介面，該介面允許您在分佈式協作環境中維護文檔的版本。

建立內容後，FrameMaker允許您以不同格式發佈文檔 — PDF、HTML5、EPUB和DITA。 您還可以執行各種檔案管理操作，如簽出、與依存對象簽出、簽入、刷新等。
使用中的FrameMaker創作 [!DNL AEM Guides] as a Cloud Service使用FrameMaker版本2020.4及更高版本。

### 新建翻譯儀表板

在Web編輯器中引入了具有以下功能的新翻譯儀表板：

* 對主題清單進行排序、搜索和篩選。
* 按參考型別篩選內容 — 直接或間接引用。
* 在啟動翻譯請求時輕鬆導航以查找現有項目。
* 引入多語言翻譯機制，以避免在針對多種語言啟動翻譯請求時為每個語言建立多個項目。
* 已引入在映射操控板中隱藏轉換頁籤的配置。 預設情況下，它是可見的。 可以選擇使用映射儀表板或Web編輯器來翻譯內容。

![翻譯儀表板](assets/translation-from-web-editor.png)

### 增強的發佈

* 作者現在可以將映射和主題級元資料傳遞到DITA-OT發佈。 當自定義PDF模板設計為使用檔案元資料屬性（如標籤、作者、文檔狀態等）時，這會很有幫助。

![DITA-OT元資料](assets/custom-meta-data-output-preset.png)

* 添加了新配置，允許用戶保留或刪除在刪除主題時 **刪除並建立** 選項用於生成AEM站點輸出。

### 改進的檔案處理

現在，在處理AEM Assets的檔案時可以看到以下改進：
* 介紹了一種新的檔案上傳體驗和選擇衝突解決策略的新對話。

![檔案上載衝突](assets/file-upload-name-conflict.png)

* 能夠建立新版本的上載檔案，並能夠防止覆蓋簽出的檔案。
* 現在，您可以直接從「版本歷史記錄」視圖中查看影像預覽。 此外，對於DITA和非DITA檔案，版本歷史記錄將分別顯示當前版本資訊。

![版本歷史記錄縮略圖](assets/version-history-preview-image.png)

* 每當用戶建立DITA檔案時，預設檔案名將顯示在小寫中，與本地資料夾建立方案AEM內聯。

### 新報告導出功能

報告在確定內容的運行狀況方面非常有用。 [!DNL AEM Guides] as a Cloud Service提供各種報告以控制您的內容。 現在，您不僅可以查看報告，還可以將報告資料導出到CSV檔案中，以查看並與您的較大團隊共用。 報告資料可讓您快速查看任何損壞的連結或丟失的影像。

![報告導出](assets/export-report.png)

### 改進的Ox DAM更新體驗

從氧氣中的服AEM務器刷新檔案時，如果當前氧氣會話中有未保存的檔案，則會顯示警告消息。 您可以選擇取消刷新操作以保存所有未保存的檔案。 如果沒有此功能，用戶將丟失其文檔中任何未保存的資訊。


### 其他功能增強

* 現在可以建立新 **Dita項目** 下 **/apps/projects/模板** 路徑。
* 現在下載預設 **ui_config_json** 資料夾配置檔案。 這可用於合併現有的自定義更改 **ui_config_json** 檔案。
* 即使存在新版本的JS檔案，也不需要清除瀏覽器快取。

## 已知問題

Adobe已確定以下已知問題 [!DNL AEM Guides] as a Cloud Service於2022年1月發佈。


### 已知問題及解決方法

對以下已知問題使用給定的解決方法：

* Web身份驗證對Mac上的Oxon連接器無效。
   **解決方法**:現在使用Windows上的氧氣接頭。

* 在Firefox瀏覽器中，如果不開啟並排視圖，則無法導入審閱注釋。
   **解決方法**:現在使用Chrome瀏覽器。

* 在移動檔案名中有空格的影像或多媒體檔案時，引用中斷。
   **解決方法**:更名檔案並在移動檔案之前從檔案名中刪除空格。

* 地圖儀表板不會在最新版本的Chrome瀏覽器中間歇性地載入。
   **解決方法**:刷新映射儀表板頁。

### 其他已知問題

* 如果氧氣連接 [!DNL AEM Guides] 使用web身份驗證的解決方案，然後註銷失敗。
* 無法將審閱任務重新分配給用戶。
* 映射集合UI中存在問題，如文本失真和 **全選** 功能無法正常工作。
