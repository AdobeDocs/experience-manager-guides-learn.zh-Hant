# 11月發行《Adobe Experience Manager指南》as a Cloud Service

## 升級到11月版

升級您當前的Adobe Experience Manager指南as a Cloud Service(稍後稱為 *導AEM線as a Cloud Service*)，請執行以下步驟：
1. 檢出Cloud Services的Git代碼，並切換到在Cloud Services管道中配置的與要升級的環境對應的分支。
1. 更新 `<dox.version>` 物業 `/dox/dox.installer/pom.xml` Cloud ServicesGit代碼的檔案2022.11.198。
1. 提交更改並運行Cloud Services管道以升級到11月版的《指南》AEMas a Cloud Service。

## 為現有內容編製索引的步驟(僅當您在9月份發佈指南之前的版本AEMas a Cloud Service時)

執行以下步驟為現有內容編製索引，並在映射級別使用新查找和替代文字：

* 向伺服器運行POST請求（使用正確的身份驗證） —  `http://<server:port>/bin/guides/map-find/indexing`。
(可選：您可以傳遞映射的特定路徑以對其進行索引，預設情況下，所有映射都將被索引 ||示例： `https://<Server:port>/bin/guides/map-find/indexing?paths=<map_path_in_repository>`)

* API將返回jobId。 要檢查作業的狀態，您可以將具有作業ID的GET請求發送到同一端點 —  `http://<server:port>/bin/guides/map-find/indexing?jobId={jobId}`
(例如：http://&lt;
_localhost:8080_>/bin/guides/map-find/indexing?jobId=2022/9/15/7/27/7dfa1271-981e-4617-b5a4-c18379f11c42_678)

* 作業完成後，上述GET請求將成功響應，並在任何映射失敗時提及。 可以從伺服器日誌中確認成功索引的映射。

## 新增功能和功能改善

指AEM南as a Cloud Service在11月版中提供了增強功能和新功能：


### 從儲存庫面板中刪除檔案

現在，您可以輕鬆地從 **選項** 的子菜單。
<img src="assets/repository-delete-file.png" alt="從儲存庫中刪除" width="500">

在刪除檔案之前，將顯示確認提示。 如果未從任何其他檔案引用該檔案，則會刪除該檔案，並顯示成功消息。

如果已簽出選定檔案，則無法將其刪除，並顯示錯誤消息。 如果所選檔案被添加到收藏夾集合或從任何其他檔案引用，AEM則參考線會檢查您的確認，並為您提供強制刪除它的選項。 如果刪除了引用的主題，並且開啟了包含要編輯的引用的檔案，則它將顯示被引用檔案的斷開連結。

**注釋**:也可以使用鍵盤的「刪除」鍵刪除所選檔案。


### 清除選定版本的檔案

在建立和維護內容時，可能會為儲存庫中的DITA檔案建立許多版本。 指AEM南允許您從儲存庫中清除舊版本的DITA檔案並釋放磁碟空間。

<img src="assets/preview-purge-report.png" alt="預覽清除報表" width="500">


參AEM考線不會刪除檔案的第一個版本，也不會刪除基線中包含的版本，或將標籤應用到該版本。 清除操作甚至不會刪除包含在翻譯或審閱工作流中的檔案。 您可以選擇要保留的版本數，也可以決定刪除早於定義天數的檔案。

在開始清除操作之前，您可以預覽報表以查看要清除的版本。 然後，您可以決定啟動或取消清除操作。

<img src="assets/download-purge-report.png" alt="PDownload清除報告" width="500">

清除操作完成後，您可以檢查清除報表以查看清除的檔案。

### 管理全局和資料夾配置檔案輸出預設

參AEM考線提供了建立和管理全局和資料夾配置檔案的輸出預設的功能。 然後，您可以輕鬆地使用這些輸出預設為與該全局或資料夾配置檔案相關的所有映射生成輸出。

<img src="assets/add-global-output-preset.png" alt="添加全局配置檔案" width="400">

**注釋** 只有資料夾級管理用戶才能建立「全局」和「資料夾配置檔案」預設。

這些全局預設顯示在 **輸出** 頁籤。 可以使用它們為所有相關映射生成輸出。 您可以選擇預設作為預設PDF預設，以生成PDF輸出。 您也可以 **編輯**。 **更名**。 **重複**&#x200B;或 **刪除** 現有輸出預設 **選項** 的子菜單。

### 「版本標籤」列已添加到翻譯儀表板

在翻譯儀表板中，您還可以看到「版本標籤」列。 這將顯示選定版本的源檔案的標籤。 這可以幫助您選擇具有特定標籤的所有檔案，並一次性翻譯它們。

<img src="assets/send-translation.png" alt="發送翻譯" width="600">


### 本地PDF |帶有顯示文檔版本差異的更改欄的PDF

現在，您可以建立一個PDF，使用更改欄顯示兩個版本之間內容的差異。 您可以選擇將當前版本與先前版本的基線進行比較，或在兩個選定的基線版本之間進行比較。

<img src="assets/pdf-change-version.png" alt="spdf-change版本" width="600">

PDF中將顯示一個更改欄，以指示已修改、插入或刪除的內容。 您還可以選擇執行以下操作：
* 以綠色和下划線顯示插入的內容
* 以紅色顯示刪除的內容，並標有刪除線

### 本地PDF |對輸出路徑和PDF檔案名的變數支援

現在，您還可以使用以下現成變數來定義「輸出路徑」和「PDF檔案」。 可以使用單個或一組變數來定義以下選項：
* `${map_filename}`
* `${map_title}`
* `${preset_name}`
* `${language_code}`
* `${map_parentpath}` （僅用於輸出路徑）
* `${path_after_langfolder}` （僅用於輸出路徑）


### 本地PDF |為DITA映射生成目錄並重新排序頁面佈局

現在，您還可以使用模板的高級PDF設定在DITA映射中生成目錄。 您可以選擇啟用或禁用各種頁面佈局的顯示，並重新排序其位置。
