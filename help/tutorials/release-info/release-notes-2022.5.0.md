# 5月發行Adobe Experience Manager指南as a Cloud Service

## 升級至5月版

升級您目前的Adobe Experience Manager指南as a Cloud Service(稍後稱為 *AEM指南as a Cloud Service*)，請執行下列步驟：
1. 查看Cloud Services的Git程式碼，並切換至Cloud Services管道中所設定的分支，該管道與您要升級的環境對應。
1. 更新 `<dox.version>` 屬性 `/dox/dox.installer/pom.xml` 檔案的Cloud ServicesGit程式碼。
1. 提交變更並執行Cloud Services管道，以升級至5月發行的AEM指南as a Cloud Service。

## 新增功能和功能改善

AEM指南as a Cloud Service在5月版本中提供許多增強功能和新功能：

### 增強的Web編輯器

* **根據自訂範本建立地圖**

現在，您擁有強大功能，可建立自訂的地圖範本。 您可以使用它們來建立DITA映射，以及映射模板中引用的主題模板和映射模板。

![dita範本](assets/dita-templates.png)

您也可以從自訂的地圖範本參考其他地圖範本和主題範本。 引用的地圖範本可以指各種地圖範本、主題範本、主題、地圖、影像、視訊和其他資產。

![建立dita範本](assets/create-dita-template.png)

自定義的映射模板可以幫助您非常輕鬆地複製映射模板和整個引用的資料夾結構。 這些自定義模板對於建立和重新建立具有遞歸結構和引用的多個映射特別有用。

* 此 **插入關鍵字** 功能已改善。 您現在可以更輕鬆地找到要插入的關鍵字，因為關鍵字是按字母順序列出的。 您也可以在「搜尋」方塊中輸入搜尋字串，以搜尋關鍵字。

![插入關鍵字](assets/insert-keyword.png)

* 現在，儲存庫檢視檔案會以批次載入。 一次載入75個檔案。 與載入資料夾中現有的所有檔案相比，此批次載入非常有效，而且您可以更快地存取檔案。

![載入更多檔案](assets/load-more-files.png)

* 您可以在所有XML編輯器畫面中呈現包含內嵌資料或連結的SVG影像，包括但不限於「預覽」和「作者」檢視。

* 預設XSD/DTD可更新為最新版本

### 改善翻譯流程

* **建立範圍翻譯專案的功能**
如果只需要建立要翻譯的項目的範圍，可以選擇 
**建立新的範圍轉換專案**. 這不會傳送要翻譯的復本，而且會保留檔案的原始翻譯狀態。

![範圍翻譯專案](assets/scoping-translation-project.png)

* 如果您拒絕翻譯工作中一個或多個主題的翻譯，則所有被拒絕主題的「正在進行」翻譯狀態將回復為其原始狀態。

* 此 **語言** 清單顯示語言資料夾及其語言代碼。 例如，法文(fr)和德文(de)。

* 翻譯功能現在也支援包含國家/地區和語言的語言代碼。 例如， `fr-fr`, `en-us`.

![翻譯語言](assets/translation-languages.png)

* 載入語言資料夾外部的DITA對應時，後端不會記錄任何例外。

如需翻譯的詳細資訊，請參閱 *從Web編輯器翻譯文檔* 區段(使用Adobe Experience Manager指南as a Cloud Service)。


### 增強發佈功能

* 您也可以存取 **發佈控制面板** 從「輸出」頁簽，從映射控制面板生成輸出。 「發佈控制面板」中提供所有作用中發佈工作的清單。

![排隊輸出](assets/queued-output.png)

* 從映射控制面板中，可以選擇多個DITAVAL檔案以生成條件化內容。 您可以新增或刪除檔案，以維護檔案順序。 您也可以將滑鼠移至檔案名稱上方，查看儲存檔案之AEM存放庫的路徑。

* **過時的功能**
AEM as a Cloud Service不再支援為FrameMaker檔案產生DITA輸出格式。 此DITA選項也已從映射控制面板的輸出預設集中移除。

### 改善文章發佈

XML編輯器提供在發佈至Salesforce設定檔時，將多個產品類別對應至文章的功能。

### 其他功能增強功能

* 預覽模式也支援 `deliveryTarget` DITA中的條件處理屬性。 它可作為下拉式篩選器中的選項，以及 **對象**, **平台**, **產品**，屬性， **其他屬性**.
* 已提供選項，可在Oxin中的AEM伺服器與本機系統之間強制同步。
