# 5月發行《Adobe Experience Manager指南》as a Cloud Service

## 升級到5月版

升級您當前的Adobe Experience Manager指南as a Cloud Service(稍後稱為 *導AEM線as a Cloud Service*)，請執行以下步驟：
1. 檢出Cloud Services的Git代碼，並切換到在Cloud Services管道中配置的與要升級的環境對應的分支。
1. 更新 `<dox.version>` 物業 `/dox/dox.installer/pom.xml` Cloud ServicesGit代碼的檔案2022.5.144。
1. 提交更改並運行Cloud Services管道以升級到5月版的指南AEMas a Cloud Service。

## 新增功能和功能改善

指南AEMas a Cloud Service在5月份發佈中提供了許多增強功能和新功能：

### 增強的Web編輯器

* **基於自定義模板建立映射**

現在，您可以獲得建立自定義地圖模板的強大功能。 可以使用它們建立DITA映射以及映射模板中引用的主題模板和映射模板。

![dita模板](assets/dita-templates.png)

也可以從自定義映射模板中參考其他映射模板和主題模板。 引用的地圖模板可以參考各種地圖模板、主題模板、主題、地圖、影像、視頻和其他資產。

![建立dita模板](assets/create-dita-template.png)

自定義的映射模板可以幫助您非常輕鬆地複製映射模板和整個引用的資料夾結構。 這些定制模板對於建立和重新建立具有遞歸結構和參照的多個映射特別有用。

* 的 **插入關鍵字** 功能已得到改進。 現在，您可以更輕鬆地找到要插入的關鍵字，因為關鍵字按字母順序列出。 也可以通過在「搜索」(Search)框中鍵入搜索字串來搜索關鍵字。

![插入關鍵字](assets/insert-keyword.png)

* 現在，在「儲存庫視圖」檔案中，將分批載入。 一次載入75個檔案。 此批處理載入是有效的，與載入資料夾中現有的所有檔案相比，您可以更快地訪問檔案。

![載入更多檔案](assets/load-more-files.png)

* 您可以在所有XML編輯器螢幕（包括但不限於「預覽」和「作者」視圖）中呈現包含嵌入資料或連結的SVG影像。

* 預設XSD/DTD可更新為最新版本

### 改進的翻譯流程

* **建立範圍轉換項目的能力**
如果只需為要翻譯的項目建立範圍，則可以選擇 
**建立新範圍界定翻譯項目**。 這不會發送要翻譯的副本，並且檔案的原始翻譯狀態會保持。

![範圍轉換項目](assets/scoping-translation-project.png)

* 如果您拒絕翻譯作業中一個或多個主題的翻譯，則所有被拒絕主題的「正在進行」翻譯狀態將恢復為其原始狀態。

* 的 **語言** 清單顯示語言資料夾及其語言代碼。 例如，法語(fr)和德語(de)。

* 現在，翻譯功能還支援包含國家和語言的語言代碼。 比如說， `fr-fr`。 `en-us`。

![翻譯語言](assets/translation-languages.png)

* 在載入語言資料夾外的DITA映射時，在後端不記錄任何異常。

有關翻譯的詳細資訊，請參閱 *從Web編輯器翻譯文檔* 的上界。


### 增強的發佈

* 您還可以訪問 **發佈儀表板** 從「輸出」(Outputs)頁籤生成映射操控板的輸出。 「發佈儀表板」中提供所有活動發佈任務的清單。

![排隊輸出](assets/queued-output.png)

* 從映射儀表板中，您可以選擇多個DITAVAL檔案以生成條件化內容。 可以通過添加或刪除檔案來維護檔案順序。 也可以將滑鼠懸停在檔案名上，查看儲存文AEM件的儲存庫中的路徑。

* **不建議使用的功能**
AEMas a Cloud Service不再支援為FrameMaker文檔生成DITA輸出格式。 此DITA選項也已從映射儀表板的輸出預設中刪除。

### 改進的基於文章的發佈

XML Editor提供了在發佈到Salesforce配置檔案時將多個產品類別映射到文章的功能。

### 其他功能增強

* 預覽模式也支援 `deliveryTarget` DITA中的條件處理屬性。 它可作為下拉篩選器中的選項，以及 **觀眾**。 **平台**。 **產品**，道具， **其他道具**。
* 已提供選項，以在Oxin中的服AEM務器與本地系統之間強制同步。
