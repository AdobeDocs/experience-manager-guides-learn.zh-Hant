# 《Adobe Experience Manager指南》2月發行as a Cloud Service

## 升級到2月版

升級您當前的Adobe Experience Manager指南as a Cloud Service(稍後稱為 *導AEM線as a Cloud Service*)，請執行以下步驟：
1. 檢出Cloud Services的Git代碼，並切換到在Cloud Services管道中配置的與要升級的環境對應的分支。
2. 更新 `<dox.version>` 物業 `/dox/dox.installer/pom.xml` Cloud ServicesGit代碼的檔案2023.2.235。
3. 提交更改並運行Cloud Services管道以升級到2月版的「指南」AEMas a Cloud Service。

## 為現有內容編製索引的步驟(僅當您在9月份發佈指南之前的版本AEMas a Cloud Service時)

執行以下步驟為現有內容編製索引，並在映射級別使用新查找和替代文字：

* 向伺服器運行POST請求（使用正確的身份驗證） —  `http://<server:port>/bin/guides/map-find/indexing`。
(可選：您可以傳遞映射的特定路徑以對其進行索引，預設情況下，所有映射都將被索引 ||示例： `https://<Server:port>/bin/guides/map-find/indexing?paths=<map_path_in_repository>`)

* API將返回jobId。 要檢查作業的狀態，您可以將具有作業ID的GET請求發送到同一端點 —  `http://<server:port>/bin/guides/map-find/indexing?jobId={jobId}`
(例如：http://&lt;
_localhost:8080_>/bin/guides/map-find/indexing?jobId=2022/9/15/7/27/7dfa1271-981e-4617-b5a4-c18379f11c42_678)

* 作業完成後，上述GET請求將成功響應，並在任何映射失敗時提及。 可以從伺服器日誌中確認成功索引的映射。

## 新增功能和功能改善

指AEM南as a Cloud Service在2月份發佈中提供了增強功能和新功能：

### 從Web編輯器生成報告

指AEM南在Web編輯器中附帶一項功能，使您能夠檢查技術文檔的整體完整性並為其生成報告。
您可以查看主題清單、管理元資料，以及查看當前映射的所有引用中使用的多媒體
**報告** 的子菜單。

**生成主題清單視圖**

您可以生成「主題清單」，該清單提供有關您的主題的詳細資訊，如參考型別、文檔狀態和作者。 您還可以生成CSV以下載DITA映射中主題的當前快照。

**管理元資料並更改文檔狀態**

您可以對單個主題應用標籤，或使用批量標籤功能對多個主題、DITA映射或子映射應用多個標籤。 也可以將所有選定主題的文檔狀態更改為下一個可能的公共文檔狀態。

<img src="assets/web-editor-metadata-panel.png" alt="管理元資料" width="600">

**生成多媒體報告**

您可以生成多媒體報告，其中包含有關當前映射中參考中使用的多媒體的詳細資訊。 您可以靈活地過濾和排序報告中列出的多媒體檔案。
您還可以生成CSV以下載DITA映射中使用的多媒體的當前快照。

<img src="assets/web-editor-reports-multimedia.png" alt="多媒體報告" width="600">

### 已修改UX的審核功能

現在，AEM指南提供了改進的UX，幫助您查看共用的供審閱的主題。 根據最新經驗，審閱功能具有以下增強功能：

* 刷新用戶介面
* 「條件」面板，允許您根據主題中的可用條件突出顯示內容
* 注釋面板中的每個注釋都連結到當前主題中的相應文本。 它可幫助您識別注釋的文本。
* 注釋按文檔中注釋文本的順序顯示。
* 審閱任務的名稱顯示在審閱工作流上。
* 選擇用於解析審閱內容中使用的所有關鍵引用和術語的審閱任務的rootmap。
* 上下文工具欄，幫助您快速突出顯示或刪除文本
* 用於編輯或刪除您自己的注釋的選項菜單
* 對於過時的注釋，您可以訪問並排視圖，該視圖可幫助您將主題的早期版本與當前審閱版本進行比較。
* 在使用過濾器時，根據選擇對右面板上的注釋進行過濾，並相應地更新左面板中的注釋數。


   <img alt="審閱任務" src="assets/comment-pop-up-panel.png" width="600">



### 翻譯增強

現在，您在「翻譯」控制板中有了更方便用戶的增強功能，可幫助您輕鬆地從Web編輯器翻譯文檔。

**將版本標籤傳遞到目標版本**

參AEM考線允許您將源檔案的標籤傳遞到目標檔案。 這有助於您輕鬆識別已轉換檔案的源版本。

<img alt="翻譯標籤" src="assets/translation-pass-source-label.png" width="600">

例如，如果您有一些源檔案已應用了版本標籤Release 1.0，則還可以將源標籤（版本1.0）傳遞給已轉換的檔案。

**強制同步不同步資產**

如果對某些資產進行更改，則「AEM參考線」將它們標籤為「不同步」。 您可以重新轉換已修改的資產或選擇取消「不同步」狀態。 例如，如果您做了一些確實不需要翻譯的小改動，則可以將其狀態標籤為「同步」。

<img src="assets/translation-version-diff.png" alt="翻譯板" width="600">

**查看主題或映射的正在轉換項目**

翻譯操控板上的某些引用可能正在進行中。 現在，AEM參考線提供了一項功能，可幫助您查看包含所選引用的所有正在進行的翻譯項目（以及目標語言）的清單。


### 從Web編輯器以各種格式生成輸出

現在，您可以從Web編輯器輕鬆生成主題或DITA映射的輸出。 您可以配置各種輸出預設AEM，如「站點」、「PDF」、「HTML5」、JSON（無頭輸出格式）和自定義輸出。 然後，可以使用這些參數生成相應的輸出。

您可以在DITA主題中定義屬性，然後使用條件預設在發佈輸出時應用條件。 您還可以使用基線發佈功能有選擇地發佈DITA映射或主題的特定版本。


### 在映射級別查找並替代文字

參AEM考線允許您在包含特定文本的映射中搜索檔案。 搜索的文本在檔案中突出顯示。 現在，您還可以在所有檔案中將搜索到的單詞或短語替換為另一個單詞或短語。 可以選擇 **全部替換** 表徵圖，以替換所有檔案中搜索到的詞的所有實例。

<img src="assets/map-find-replace.png" alt="映射查找替換" width="600">

### 從儲存庫面板中刪除和複製檔案

現在，您可以輕鬆地從 **選項** 的子菜單。 預設情況下，檔案是使用尾碼建立的(如 `filename_1.extension`)。

<img src="assets/options-menu-repo-view-file-level.png" alt="檔案選項菜單 " width="500">


### 其他Web編輯器增強

* 在「AEM參考線」中，可以使用上下文菜單對影像和媒體檔案執行一些常見操作。 現在，您還可以在儲存庫中找到選定的影像或介質，或在「資產」UI中查看檔案的預覽。

* 當前資料夾配置檔案的名稱將作為主工具欄中「用戶首選項」表徵圖的標籤顯示。 這有助於您確定正在處理的資料夾配置檔案。

* 在映射視圖中開啟映射時，當前映射的標題將顯示在主工具欄的中心。 這有助於讓用戶知道當前開啟的地圖。


### 在氧氣編輯器中查看標題以代替UUID

現在AEM參考線允許您選擇 **在編輯器和映射管理器中使用標題** 的子菜單。 如果選擇此選項，則在編輯器或DITA映射管理器中開啟時，檔案的標題將顯示在檔案的頁籤上。 如果未選擇此選項，則檔案的UUID將顯示在檔案的頁籤上。

### 基於微服務的指南AEMas a Cloud Service發佈

新的發佈微服務使您能夠在指南as a Cloud Service上同時運行大AEM型發佈工作負載，並利用業界領先的Adobe I/O Runtime無伺服器平台。

對於每個發佈請AEM求，指南as a Cloud Service運行一個單獨的容器，該容器根據用戶請求水準擴展。 這使您能夠運行多個發佈請求並獲得改進的效能。

有關詳細資訊，請參閱 [為指南as a Cloud Service配置新的基於微AEM服務的發佈](https://experienceleague.adobe.com/docs/experience-manager-guides-learn/tutorials/knowledge-base/publishing/configure-microservices.md)。

### 本地PDF |在PDF輸出中添加自定義書籤

現在，您可以在最終PDF輸出中的特定內容上添加自定義書籤，以便輕鬆導航。 這將添加到根據DITA映射中的主題或節標題建立的目錄中。

### 本地PDF |對目錄條目和主題內容應用自定義樣式

參AEM考線提供了將自定義樣式應用於目錄條目或PDF輸出中的特定主題的功能。 例如，可以更改目錄和主題標題中文本的顏色。 您還可以對主題內的整個內容應用樣式。


### 本地PDF |在腳注元件中設定頁面標籤樣式

現在，您可以在腳注中設定頁面標籤樣式。 例如，可添加括弧或更改其顏色。 這些樣式可幫助用戶輕鬆識別文檔中的頁面標籤。

### 本地PDF |更改欄以指示目錄中更改的主題

現AEM在，參考線允許您快速確定PDF輸出目錄中更改的主題。  它顯示目錄中更改主題左側的更改欄。 可以按一下目錄中的主題並查看詳細更改。

<img src="assets/change-marker-toc.png" alt="更改目錄中的標籤 " width="500">

## 已修正的問題

以下列出了在不同區域修復的錯誤：

### 編寫

* Web編輯器html中的更改會導致 `<dl>` 和 `<dlentry>`。 (11024)
* 某些屬性未被視為條件並導致問題。 (10895)
* 三個或更多級別嵌套 `<indexterm>` 未嵌套在本機PDF導出中。 (10799)
* 內容將消失在從「作者」視圖切換到「源」視圖的任務正文中。 (10735)
* 審閱備注在審閱任務中錯放。 (10625)
* **撤消** 或 **重做** 在某些檔案上無法正常工作。 (10373)
* 未在複製和貼上操作中保留自定義元資料。 (10367)
* XML編輯器中的「撤消」選項將用戶帶到頁面頂部。 (10091)
* 在資產的複製和貼上操作後，將刪除節點屬性。 (10053)
* MIMEType硬編碼用於DITA資產建立和更新。 (8979)
* 版本歷史記錄中的版本建立者名稱是「fmdita-serviceuser」，用於通過資產用戶介面上載的檔案。 (8910)
* 在雲上安裝輔助線時，無AEM法複製和貼上內容片段。 (11315)
* 瀏覽器（Web編輯器）在使用自定義架構載入內容時凍結。 (11211)

### 管理

* 複製DITA映射資產（從資產UI）會導致複製資產中出現錯誤的基線。 (11218)
* 上載的檔案大於（預設為2 GB）中允許的限制時，不會顯示警AEM告消息。 (10817)
* Web編輯器 — 基線 | Web編輯器中新基線儀表板中「最新」列的行為不同。 (10808)
* 翻譯 |由於/libs/fmdita/i18n/ja.json無效，翻譯作業未開始。 (10543)
* 翻譯 |在從翻譯儀表板（人工翻譯）建立的範圍轉換項目中出錯。 (10526)
* 翻譯 |對活動翻譯項目中存在資產的整個語言資料夾阻止後處理。 (10332)
* 如果版本已更改並保存在「基線」編輯器中，則任何資產都會出現多個彈出窗口。 (10399)
