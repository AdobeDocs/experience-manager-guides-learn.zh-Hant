# 了解Web編輯器功能 {#id176NC500V5Z}

本節將逐步帶您了解Web編輯器中提供的各種功能。 我們可將網頁編輯器分為下列區段或區域：

- [主工具欄](#id2051EA0G05Z)
- [輔助工具欄](#id2051EA0J0Y4)
- [左側面板](#id2051EA0M0HS)
- [內容編輯區域](#id2051EB000UI)
- [右面板](#id2051EB003YK)

以下小節詳細介紹Web編輯器的各個部分。

## 主工具欄 {#id2051EA0G05Z}

主工具欄位於Web編輯器介面的頂部，它提供Web編輯器中提供的檔案級功能和各種創作模式。 頂端工具列中可用的功能說明如下：

**儲存全部** - ![](images/SaveFloppy_icon.svg)

儲存您在所有已開啟的主題中所做的變更。 如果您在網頁編輯器中開啟了多個主題，請按一下 **全部儲存** 或使用 **Crtl**+**S** 按一下即可保存所有文檔。 您不必單獨保存每個文檔。

>[!NOTE]
>
> 「儲存」操作不會建立新版本的主題。 要建立新版本，請選擇另存為新版本。

**另存為新版本** - ![](images/save-revision-icon.png)

保存您在主題中所做的更改，並建立新的主題版本。 如果您正在處理新建立的主題，版本資訊將顯示為 **無**.

![](images/save-all-first-version-none_cs.png){width="800" align="left"}

版本編號會隨著為主題或映射檔案建立的每個新版本而改變。

當您選擇使用 **另存為新版本**，則會出現下列對話方塊：

![](images/save-as-new-version-dialog.PNG){width="300" align="left"}

輸入注釋和版本標籤以標識更改，然後按一下 **儲存** 來建立新版本的檔案。

當您選擇 *另存為新版本*，則主題的第一版會在DAM中建立，而此版本也會成為您目前使用中的主題版本。 稍後，如果您回復成舊版本的主題，則該主題將成為您目前使用中的主題版本。

如果您的管理員已預先設定版本標籤，您會在下拉式清單中看到這些標籤。 您可以從可用標籤清單中選擇標籤並保存文檔。

![](images/web-editor-pre-defined-labels.PNG){width="300" align="left"}

在保存主題時，可以添加一條注釋，指定您在主題中所做的更改。 此注釋顯示在主題的「版本歷史記錄」中。

如果您的主題正在審核中，審核者會收到通知，告知主題有較新版本可用。 他們可以輕鬆訪問文檔的最新修訂版本，並繼續審閱主題的最新版本。

將指標暫留在主題標題上時，會顯示檔案路徑和版本號。

![](images/mouse-hover-on-title_cs.png){width="800" align="left"}

>[!NOTE]
>
> 主題版本可用後，您也可以將標籤新增至主題。 然後，這些標籤可用於建立用於發佈特定版本的文檔的基線。 如需在主題中使用標籤的詳細資訊，請參閱 [使用標籤](web-editor-use-label.md#).

**還原和重做** - ![](images/Undo_icon.svg) / ![](images/Redo_icon.svg)

還原或重做最後一個動作。

**刪除元素** - ![](images/Delete_icon.svg)

刪除當前選定的元素或游標所在的元素。

**查找和替換** - ![](images/FindAndReplace_icon.svg)

「查找和替換」功能在「作者」和「源」視圖模式中可用。 「查找和替換」(Find and Replace)文本欄出現在主題編輯區域的底部。 您可以使用快捷鍵 **CTRL**+**F** 調用「查找和替換」欄。

![](images/find-replace-bar.png){width="800" align="left"}

使用設定圖示\(![](images/settings-find-replace-icon.svg)\)，您可以切換 **忽略大小寫** 和 **僅全字** 搜尋選項。 若要執行不區分大小寫的搜尋，請開啟\（或選取\） **忽略大小寫** 選項。 否則，如果要執行區分大小寫的搜索，請關閉\（或取消選擇\） **忽略大小寫** 選項。 您也可以選擇搜尋整個字。

搜尋是即時的，這表示當您在 **查找** 欄位中，會立即搜尋並選取主題中的詞語。 同樣地，若要取代主題中的文字，請在個別欄位中輸入搜尋詞及其取代，然後按一下 **取代** 或 **全部替換** 按鈕。

在「源」視圖中，「查找和替換」對於搜索特定元素或屬性非常有用。 例如，如果您要取代 `@product` 屬性，則可從「來源」檢視輕鬆完成。 「作者」檢視不允許您根據屬性或元素進行搜尋。 不過，在使用 **全部替換** 功能，因為可能會覆寫XML程式碼。

**編輯器設定** - ![](images/editor_settings_icon.svg)

編輯器設定僅供管理使用者使用。 管理員可以使用首選項配置以下設定：

>[!NOTE]
>
> 如果要更新任何預設設定，則應重新開啟文檔以使更改生效。

- **一般**:「一般」設定可讓您將字典設定為與Web編輯器搭配使用。 此索引標籤包含三個區段： **拼字檢查**, **條件**，和 **製作**.

   ![](images/editor-setting-general.png){width="650" align="left"}

   - **拼字檢查**:有兩種選擇 —  **AEM拼字檢查** 和 **瀏覽器拼字檢查**. 依預設，編輯器會使用「瀏覽器拼字檢查」功能，其中拼字檢查是使用瀏覽器的內建字典來執行。 您可以切換至AEM Spell Check，使用AEM字典，這也可以自訂，以新增自訂字詞清單。 如需自訂AEM字典的詳細資訊，請參閱 *自訂AEM預設字典* 一節中的「安裝及設定Adobe Experience Manager指南」as a Cloud Service。


   - **條件**

      - **在「作者」檢視中反白顯示條件文字**:選取此選項，以反白標示製作檢視中的條件文字。 條件內容會使用為條件定義的顏色加亮顯示。

      - **使用條件屬性進行驗證**:選擇此選項可允許驗證為屬性定義的值。 這可防止您新增任何不正確的值。

      - **在「主旨配置」面板中顯示帶標題的鍵**:選擇此選項可顯示主題方案中的鍵和標題。 如果您未選取此選項，則只會顯示標題。 例如，此處也會顯示鍵「os」、「audience」和「other」，以及標題。

         ![](images/subject-scheme-title.png){width="550" align="left"}

      - **在「條件」面板中顯示主體配置**:選取此選項，即可在「條件」面板中查看主旨配置。 如果您取消選取此選項，定義的條件會顯示在條件面板中。
   - **編寫**

      - **啟用全部替換**:選擇此選項可查看「查找和替換」面板中的「全部替換」表徵圖。


**面板**:此設定會控制顯示在編輯器左側面板中的面板。 您可以切換開關以顯示或隱藏所需面板。

![](images/editor-setting-panel.png){width="650" align="left"}

>[!NOTE]
>
> 如果已設定自訂面板，該面板也會出現在面板清單中。 您可以切換開關以顯示或隱藏自訂面板。 如需設定的詳細資訊，請參閱 *在左側面板中設定自訂面板* 一節中的「安裝及設定Adobe Experience Manager指南」as a Cloud Service。

- **元素清單**:身為管理員，您可以控制製作者可使用 [插入元素](#id204SG30105Z) 快顯視窗，並定義元素的顯示名稱。 元素清單設定允許您根據DITA規範指定元素的名稱，以及您要使用的標籤，而不是DITA定義的元素名稱：

   ![](images/editor-setting-element-list.png){width="650" align="left"}

在上方的螢幕擷取中， `b` 元素的標籤為粗體， `codeblock` 會連同一些其他元素一起提供代碼區塊標籤。 如果您選取 **僅使用以上元素** 選項，則「插入元素」彈出式視窗中只會顯示此清單中的有效元素\（位於目前的插入點\）。

在以下螢幕擷取中，前一螢幕擷取中的4個已設定元素中，只有3個會顯示在目前的內容中：

![](images/editor-setting-insert-element-list.PNG){width="300" align="left"}

- **屬性清單**:與「元素清單」類似，您可以控制要在元素的屬性清單中顯示的屬性清單及其顯示名稱。 在以下螢幕擷取中，只有3個屬性已設定為顯示在元素的屬性清單中：

![](images/editor-setting-attributes-list.png){width="650" align="left"}

使用此設定時，當您嘗試將屬性新增至元素時，您只會看到清單中設定的屬性清單。

![](images/editor-setting-add-attributes-list.png-to-element.PNG){width="300" align="left"}

- **顯示屬性**:與「屬性清單」一樣，您可以控制要顯示在元素的屬性清單中的屬性清單。 預設為4 **顯示屬性**  — 對象、平台、產品和prop已設定為顯示在元素的屬性清單中。 您也可以使用 **新增** 表徵圖。 您也可以使用 **刪除** 表徵圖。

為元素定義的屬性顯示在「佈局」和「大綱」視圖中。

![](images/editor-settings-display-attributes.png){width="550" align="left"}

- **翻譯**:此頁簽包含將源標籤傳播到目標版本的選項。

   - **將源版本標籤傳播到目標版本**:選擇此選項可將源檔案版本的標籤傳遞到翻譯的檔案。 依預設，此功能會停用。

   ![](images/editor-setting-translation.png){width="550" align="left"}


**使用者偏好設定** - ![](images/user_preference_editor_icon.svg)

所有作者皆可使用「使用者偏好設定」。 使用偏好設定，作者可以設定下列設定：

![](images/user_preference_editor.PNG){width="550" align="left"}

- **主題**:您可以從編輯器的「淺色」、「最淺色」、「深色」或「最深」主題中選擇。 對於Lightest主題，工具欄和面板使用更淺的灰色背景。 對於「淺」主題，工具欄和面板使用淺灰色背景。 在「最暗」主題中，工具欄和面板使用更深的黑色背景。 如果是深色主題，工具欄和面板使用黑色背景。 在所有主題中，內容編輯區域以白色背景顯示。

- **資料夾描述檔**:「資料夾配置檔案」控制與條件屬性、創作模板、輸出預設集和Web編輯器配置相關的各種配置。 預設會顯示「全域設定檔」。 此外，如果管理員已在系統中配置了資料夾配置檔案，則這些資料夾配置檔案也會顯示在「資料夾配置檔案」清單中。

   管理員可在資料夾設定檔中定義的Web編輯器設定包括：自訂使用者介面，包括工具列圖示、網頁編輯器的版面、片段和根圖。 如需詳細資訊，請參閱 *配置全局或資料夾級配置檔案* (位於「安裝及設定Adobe Experience Manager指南」as a Cloud Service)。

   >[!NOTE]
   >
   > 當前「資料夾配置檔案」的名稱將作為主工具欄中「用戶首選項」表徵圖的標籤顯示。

- **基本路徑**:依預設，當您從Web編輯器存取AEM存放庫時，會顯示/content/dam位置中的資產。 您的工作資料夾很可能是/content/dam/資料夾內的一些資料夾。 您每次只需按幾下，即可抵達工作資料夾。 您可以設定工作資料夾的基本路徑，然後設定儲存庫視圖，然後提前顯示該位置的內容。 這可縮短存取工作資料夾的時間。 此外，當您在主題中插入任何引用或媒體檔案時，檔案瀏覽位置將從基本路徑中設定的資料夾開始。

- **選擇根映射**:選擇DITA映射檔案以解析密鑰引用或術語表項。 所選根映射的優先順序最高，以解析鍵引用。 如需詳細資訊，請參閱 [解析關鍵引用](map-editor-other-features.md#id176GD01H05Z).


>[!NOTE]
>
> 如果您不想使用任何根對應，請確定 **選擇根映射** 欄位空白。

**製作、來源和預覽模式**

有關各種創作和文檔查看模式的詳細資訊，請參見 [Web編輯器檢視](web-editor-views.md#).

## 輔助工具欄 {#id2051EA0J0Y4}

在Web編輯器中開啟要編輯的主題時，輔助工具欄將出現。 次要工具列中可用的功能說明如下：

**插入元素** - ![](images/Add_icon.svg)

在當前或下一個有效位置插入有效元素。 如果您在區塊元素內工作，如 `note`，然後使用「插入元素」圖示，在 `note` 元素。 在以下螢幕擷取中，已將附註元素插入p \(paragraph\)元素內：

![](images/note-in-para-insert-element_cs.png){width="800" align="left"}

如果在注釋元素中按Enter鍵，則在注釋元素本身中會建立新段落。 若要在附註外插入新元素，請在元素階層連結中按一下p元素\（螢幕擷取畫面中反白顯示\），然後按一下「插入元素」圖示或按 ***Alt***+***輸入*** 開啟「插入元素」快顯視窗。 然後，選取所需元素，然後按Enter鍵，將選取的元素插入在附註元素之後。

當閃爍的塊游標出現時，您也可以在兩個元素之間添加元素。

![](images/Block-cursor.png){width="300" align="left"}

例如，如果您正在處理DITA主題，且區塊游標在簡短說明和內文之間閃爍，則可以新增 `prolog` 元素，然後新增版權、作者和其他詳細資訊。

輸入新元素的另一種方式是使用內容功能表。 按一下右鍵文檔中的任何位置以調用上下文菜單。 從此菜單中選擇「插入元素」以顯示「插入元素」對話框，並選擇要插入的元素。

![](images/insert-element-before-after.png){width="300" align="left"}

**插入段落** - ![](images/Paragraph_icon.svg)

在當前或下一個有效位置插入段落元素。

**插入/移除編號清單** - ![](images/TextNumbered_icon.svg)

在當前或下一個有效位置建立編號清單。 如果您在編號清單上並按一下此表徵圖，則該項目將轉換為普通段落。

**插入/移除項目符號清單** - ![](images/BulletList_icon.svg)

在當前或下一個有效位置建立項目符號清單。 如果在項目符號清單上並按一下此表徵圖，則項將轉換為普通段落。

**插入表格** - ![](images/Table_icon.svg)

在當前或下一個有效位置插入表。 按一下「插入表」表徵圖以開啟「插入表」對話框：

![](images/table-properties.png){width="550" align="left"}

您可以指定表格中需要的列數和欄數。 如果要將第一行保留為表標題，請選擇「將第一行設定為標題」選項。 若要將標題新增至表格，請在「標題」欄位中輸入標題。

插入表格後，可以使用上下文菜單修改表格。

![](images/table-context-menu_cs.png){width="550" align="left"}

使用表格的上下文菜單，您可以：

- 插入儲存格、列或欄

- 向右和向下合併單元格

- 水準或垂直分割儲存格

- 刪除儲存格、列或欄

- 從表格建立程式碼片段

- 產生ID


您也可以在表格的多個儲存格、整列或欄上定義屬性。 例如，若要對齊表格儲存格，請拖曳並選取所需的儲存格。 在「內容屬性」面板\（位於右側）中，屬性 **類型** 變更 **多項**. 在「其他屬性」區段中，選取 `@valign` 屬性。 從「值」下拉式清單中，選取您要對選取的表格儲存格套用的所需文字對齊方式。

![](images/align-table-cell_cs.png){width="800" align="left"}

**插入影像** - ![](images/Image_icon.svg)

在當前或下一個有效位置插入影像。 按一下「插入影像」圖示以開啟「插入影像」對話方塊，然後搜尋並選取您要插入的影像。

>[!NOTE]
>
> 您也可以將影像從本機系統拖放至文章，以新增影像。 在此情況下，會使用 **上傳資產** 工作流程。  如需詳細資訊，請參閱 **上傳資產** 工作流程 [左面板](web-editor-features.md#id2051EA0M0HS) 區段。


![](images/insert-image.png){width="650" align="left"}

可以在「插入影像」對話框中為影像添加影像/表徵圖題和替代文本。

您可以在頂端的「要搜尋的類型」列中輸入檔案名稱，以搜尋所需的影像檔案，也可依路徑\（要在\中搜尋）、集合、檔案類型和標籤來篩選搜尋結果。 找到所需的影像檔案後，選擇該檔案並按一下「選擇」以在文檔中插入影像。 您可以插入各種格式的影像檔案，例如 `.png`, `.svg`, `.gif`, `.jpg`, `.eps`, `.ai`, `.psd`、等。

插入影像後，您可以從「內容屬性」面板更改高度、寬度、位置和屬性。 按一下影像檔案，然後在右側邊欄的「內容屬性」面板中進行變更。

![](images/image-properties.png){width="800" align="left"}

「源」欄位顯示插入的影像檔案的UUID。 通過將滑鼠指針懸停在「源」欄位上，可以找到插入的影像檔案的完整路徑。 路徑會顯示在工具提示中。

您可以為影像檔案提供「高度」或「寬度」值，以調整影像大小。 自動保持影像的長寬比。 如果需要，也可以通過按一下鎖定表徵圖\（「維護外觀比例」\）並提供「高度」和「寬度」值來選擇不維護影像檔案的外觀比例。

也可以將影像的「放置」(Placement)設定指定為「內嵌」(Inline)或「插入」(Break)。 如果選擇使用「斷開」放置選項，則可以選擇將影像對齊的位置\（左、中或右\）。

您也可以選取 **屬性** 欄位。

>[!NOTE]
>

**合併** - ![](images/merge-icon.svg)

當您在多作者環境中工作時，就很難追蹤其他作者在主題或地圖中所做的變更。 「合併」功能不僅讓您能夠查看更改，還能夠更好地控制在最新版本的文檔中保留哪些更改。

**合併主題檔案**

要合併主題中的更改，請執行以下步驟：

1. 在網頁編輯器中開啟主題。

1. 按一下 **合併**.

   將出現「合併」對話框。

   ![](images/merge-changes-in-topic.png){width="550" align="left"}

1. *\（可選\）* 您也可以從存放庫中的其他位置瀏覽並選取新檔案。

1. 選擇要與當前版本的檔案進行比較的檔案版本。

1. 從「選項」中選擇：

   - **從選取的版本追蹤變更**:此選項會以追蹤變更的形式顯示所有內容更新。 然後，您可以選擇一次接受或拒絕文檔中的更改，或一次全部接受更改。

   - **回復到所選版本**:此選項將文檔的當前版本還原為選定版本。 此選項不會讓您控制接受或拒絕的內容。

1. 按一下 **完成**.

1. 如果您選取 **追蹤從選取的版本變更** 選項，則來自選定版本的所有更改將顯示在右側面板的「跟蹤更改」功能中。

   您可以選擇接受或拒絕「跟蹤的更改」面板中的所有注釋，或接受或拒絕單個注釋。


**合併映射檔案**

要合併映射檔案中的更改，請執行以下步驟：

1. 在網頁編輯器中開啟地圖。

1. 按一下 **合併**.

   將出現「合併」對話框。

   ![](images/merge-changes-in-map.png){width="550" align="left"}

1. *\（可選\）* 您也可以從存放庫中的其他位置瀏覽並選取新檔案。

1. 選擇要與當前版本的檔案進行比較的檔案版本。

1. 從「選項」中選擇：

   - **從選取的版本追蹤變更**:此選項會以追蹤變更的形式顯示所有內容更新。 然後，您可以選擇一次接受或拒絕文檔中的更改，或一次全部接受更改。

   - **回復到所選版本**:此選項將文檔的當前版本還原為選定版本。 此選項不會讓您控制接受或拒絕的內容。

1. 按一下 **完成**.

   1. 如果您選取 **追蹤從選取的版本變更** 選項，則選定版本的所有更改將顯示在「跟蹤的更改」面板\（位於右側\）中。

      您可以選擇接受或拒絕「跟蹤的更改」面板中的所有更改，或接受或拒絕映射檔案中的單個更改。

**版本標籤管理** -  ![](images/version-label-icon.svg)

標籤可幫助您確定DDLC \（文檔開發生命週期\）中特定主題所在的階段。 例如，當您處理主題時，可將標籤設為「已核准」。 一旦發佈主題並可供客戶使用後，您就可以將「已發行」標籤指派給該主題。

AEM指南可讓您以自由格式文字格式指定標籤，或使用一組預先定義的標籤。 自訂標籤可讓系統中的任何作者根據其選擇指定標籤。 這給了靈活性；但是，這會在系統中引入不一致的標籤。 為克服此問題，管理員可設定一組預先定義的標籤。 如需設定預先定義標籤的詳細資訊，請參閱 *配置和自定義XML Web編輯器* (位於「安裝及設定Adobe Experience Manager指南」as a Cloud Service)。

這些標籤會以下拉式清單的形式顯示給作者，供他們指定標籤的位置使用。 這可確保系統中只使用預先定義的一致標籤。

您可以透過不同的方法將標籤套用至主題 —  [版本記錄](web-editor-use-label.md#) 面板， [基線](/help/tutorials/user-guide/generate-output-use-baseline-for-publishing.md#id184KD0T305Z) UI和Web編輯器。 網頁編輯器中的版本標籤功能可讓作者快速輕鬆地將標籤指派給其主題。

要從Web編輯器向主題添加標籤，請執行以下步驟：

1. 在網頁編輯器中開啟主題。

1. 按一下 **版本標籤**.

   此時將顯示「版本標籤管理」對話框。

   ![](images/version-label-management-dialog.png){width="650" align="left"}

   「版本標籤管理」對話框分為兩個部分 — 左側面板有可用於該主題的版本清單以及標籤下拉清單\（或輸入標籤的文本框\），右側面板預覽該主題。

1. 選取要套用標籤的版本。

   當您從版本清單中選擇主題的不同版本時，預覽面板將顯示當前版本與主題的選定版本之間的更改

   >[!NOTE]
   >
   > 如果已對版本套用標籤，則標籤會顯示在下拉式清單中版本號碼旁邊、選取版本清單下。 您可以按一下「\」(**x**\)圖示。

1. 如果您的管理員已定義標籤清單，則會顯示標籤的下拉式清單，您可從中選擇要套用的標籤。 您可以從下拉式清單中選取多個標籤。

   否則，將顯示一個文本框，您可在其中輸入要添加到主題的標籤。

   >[!NOTE]
   >
   > 您無法對主題的多個版本套用相同的標籤。 如果您嘗試關聯現有標籤，則會提供一個選項，可將其從現有版本中刪除，並將其應用於主題的選定版本。

1. 按一下 **添加標籤**.

1. 在「套用標籤」確認訊息中，選取 **移動標籤** 選項，將標籤從現有版本移至所選版本。 如果您未選擇此選項，並且有標籤被分配給主題的不同版本，則標籤不會移至所選主題的版本。 在標籤應用程式中會忽略這些標籤。

- **主題**:選擇 **目前主題** 在網頁編輯器中開啟，全部 **已開啟的主題** ，或 **目前地圖** 在地圖編輯器中編輯以搜尋詞語。
- **依狀態篩選主題**:您可以選擇將搜索限制為具有選定文檔狀態的主題。 主題可以是草稿、編輯、審核中、批准、審核、完成狀態，或組織設定的任何狀態。
- **動作**:您可以選擇搜索辭匯表鍵 **手動處理每個主題** 或 **對所有主題自動**. 如果您選擇 **手動處理每個主題**，在轉換每個主題中的每個詞語之前，會提示您確認。 如果您選擇 **對所有主題自動**，它會自動轉換所有主題中的所有詞語。
- **轉換**:您可以轉換已搜尋的 **文本到術語** 或 **辭匯表術語到文本。**
- **選項**:您可以選取下列選項：
   - **區分大小寫比對**:搜尋詞語，以尋找大小寫相同的相符項目。 例如，「USB」與「usb」不匹配。
   - **僅轉換第一個例項**:如果主題中出現多個搜尋詞例項，則只會轉換第一個例項。
   - **轉換前簽出檔案**:在轉換詞語之前，將簽出搜索的檔案。
   - **轉換後建立新版本**:轉換完成後，會建立新版本的主題。
- **下一個** 按鈕 **手動處理每個主題** 選項。 按一下 **下一個** 根據所選設定轉換每個主題的詞語。 它會提示每個主題中的詞語轉換，並移至下一個檔案。 您可以選擇轉換詞語或略過詞語，然後移至下一個詞語。

   ![](images/manual-convert-skip.png){width="300" align="left"}

- **轉換** 按鈕 **對所有主題自動** 選項。 選擇 **轉換** 將文檔中找到的所有術語轉換為連結的辭匯縮寫。

清單 **已更新主題** 以及轉換後的詞語和 **有錯誤的主題** 的下界。 暫留在\( ![](images/info-icon.svg)\)「主題有錯誤」旁的圖示，以查看錯誤的詳細資訊。

AEM指南也支援巢狀層級的主旨配置圖，而且您可以在根主旨配置圖下定義多個主旨配置。

以下範例說明如何在AEM指南中使用主旨配置。

1. 在您選擇的工具中建立主題配置檔案。 以下XML代碼建立綁定 `platform` 屬性。

   ```XML
   <?xml version="1.0" encoding="UTF-8"?>
   <!DOCTYPE subjectScheme PUBLIC "-//OASIS//DTD DITA Subject Scheme Map//EN" "subjectScheme.dtd">
   <subjectScheme id="GUID-4f942f63-9a20-4355-999f-eab7c6273270">
       <title>rw</title>
       <!-- Define new OS values that are merged with those in the unixOS scheme -->
       <subjectdef keys="os">
           <subjectdef keys="linux">    </subjectdef>
           <subjectdef keys="mswin">    </subjectdef>
           <subjectdef keys="zos">    </subjectdef>
       </subjectdef>
       <!-- Define application values -->
       <subjectdef keys="app" navtitle="Applications">
           <subjectdef keys="apacheserv">    </subjectdef>
           <subjectdef keys="mysql">    </subjectdef>
       </subjectdef>
       <!-- Define an enumeration of the platform attribute, equal to       each value in the OS subject. This makes the following values       valid for the platform attribute: linux, mswin, zos -->
       <enumerationdef>
           <attributedef name="platform">    </attributedef>
           <subjectdef keyref="os">    </subjectdef>
       </enumerationdef>
       <!-- Define an enumeration of the otherprops attribute, equal to       each value in the application subjects.       This makes the following values valid for the otherprops attribute:       apacheserv, mysql -->
       <enumerationdef>
           <attributedef name="otherprops">    </attributedef>
           <subjectdef keyref="app">    </subjectdef>
       </enumerationdef>
   </subjectScheme>
   ```

   > 若要啟用 **全部替換** 表徵圖，您的系統管理員必須選擇 **啟用全部替換** 在 **一般** 標籤 **編輯器設定**.


在整個系統中一次只能執行一次替換所有操作，在執行操作之前，您將看到「替換所有正在進行的操作」狀態。 您也可以中止中的取代所有操作，或查看記錄報表。 如果中止操作，您會在收件匣中收到有關該操作的通知。 替換選定檔案中的所有出現次數後，將顯示成功通知。

![](images/replace-all-in-progress.png){width="400" align="left"}

您也可以使用 **在地圖中查找** 選項 **選項** 尋找和取代地圖中文字的地圖功能表。 對於在儲存庫面板中或在地圖視圖中開啟的地圖，將顯示此選項。

![](images/map-options-menu.png){width="550" align="left"}

## 內容編輯區域 {#id2051EB000UI}

內容編輯區域是顯示主題或地圖內容的位置。 您可以在此區域中編輯所有內容。 它提供您所編輯內容的WYSIWYG檢視。 您可以同時開啟多個主題，這些主題會顯示在其各自的標籤中。 在檔案的標籤下方，元素的階層連結位於目前的游標位置。 在內容編輯區域的右上角，會顯示目前主題的版本號。

![](images/content-editing-area.png){width="650" align="left"}

## 右面板 {#id2051EB003YK}

右面板是永久性面板，其中包含有關當前所選文檔的資訊。

>[!NOTE]
>
> 右側面板可調整大小。 要調整面板大小，將游標置於面板邊界上，游標將更改為雙頭箭頭，按一下並拖動以調整面板寬度。

右側面板可讓您存取下列功能：

**內容屬性** -  ![](images/content-properties-icon.svg)

您可以按一下右側面板中的「內容屬性」圖示，以存取「內容屬性」功能。 「內容屬性」面板包含有關文檔中當前所選元素類型及其屬性的資訊。 您也可以從下拉式清單中選取屬性並指定屬性的值，以新增屬性。

>[!NOTE]
>
> 即使您的主題包含參考的內容，您也可以使用屬性面板在其上新增屬性。

如果您的管理員已建立屬性的設定檔，則您會取得這些屬性及其設定的值。 使用內容屬性面板，您可以選擇這些屬性並將它們指派給主題中的相關內容。 這樣，您也可以建立條件式內容，然後可用來建立條件式輸出。 如需使用條件預設集產生輸出的詳細資訊，請參閱 [使用條件預設集](generate-output-use-condition-presets.md#).

![](images/properties-tab-attributes_cs.png){width="300" align="left"}

**檔案屬性** -  ![](images/topic-properties-icon.svg)

按一下右側面板中的「檔案屬性」表徵圖，查看所選檔案的屬性。 「檔案屬性」(File Properties)具有以下兩個部分：

**一般**

「一般資訊」區段可讓您存取下列功能：

![](images/file-properties-general.png){width="300" align="left"}

- **名稱**:顯示所選主題的檔案名。 檔案名稱超連結到所選檔案的屬性頁。
- **ID**:顯示所選主題的ID。
- **中繼資料標籤**:這些是主題的中繼資料標籤。 它們是從屬性頁面中的標籤欄位中設定。
- **語言**:顯示主題的語言。 它是從屬性頁面中的語言欄位設定。
- **建立於**:顯示建立主題的日期和時間。
- **簽出者**:顯示已簽出主題的用戶。
- **檔案狀態**:您可以選擇和更新當前開啟的主題的文檔狀態。 如需詳細資訊，請參閱 [檔案狀態&#x200B;](web-editor-document-states.md#)*.*

**注意：** 您可以將檔案屬性中各欄位的屬性值複製到剪貼簿。

**引用**

「參考」(References)部分允許您訪問以下功能：

![](images/file-properties-references.png){width="300" align="left"}

- **用於**:「在引用中使用」(Used In references)列出引用或使用當前檔案的文檔。
- **傳出連結：** 「傳出連結」列出當前文檔中引用的文檔。

將滑鼠移至檔案的參考上，即可在工具提示中取得檔案路徑和檔案的UUID。

**注意：** 所有在和傳出引用都超連結到文檔。 您可以輕鬆開啟和編輯連結的文檔。

除了開啟檔案外，您也可以使用 **選項** 菜單。 您可以執行的某些動作包括編輯、預覽、複製UUID、複製路徑、新增至我的最愛、屬性和開啟地圖控制面板。

**檢閱** -  ![](images/review-icon.svg)

按一下「審閱」表徵圖將開啟審閱面板，您可以在其中為當前開啟的文檔建立審閱任務。

![](images/review-panel-before-opening.png){width="300" align="left"}

如果已建立多個審閱項目，則可以從下拉清單中選擇一個項目並訪問審閱注釋。

使用「檢閱」面板，您可以檢視並張貼對主題上所提供留言的回覆。 您可以逐一接受或拒絕評論。

如需詳細資訊，請參閱 [地址審核注釋](review-address-review-comments.md#).

**追蹤的變更** -  ![](images/track-change-icon.svg)

使用右側面板的「跟蹤的更改」功能，可以查看文檔中所有更新的資訊。 您也可以搜尋對檔案所做的任何特定更新。

>[!NOTE]
>
> 「追蹤的變更」功能會顯示使用主工具列的「啟用/停用追蹤變更」功能來追蹤的所有更新。 如需詳細資訊，請參閱 [啟用/停用追蹤變更](#id205DF0203Y4).
