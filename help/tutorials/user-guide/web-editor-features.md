# 瞭解Web編輯器功能 {#id176NC500V5Z}

本節將介紹Web編輯器中提供的各種功能。 我們可以將Web編輯器分為以下部分或區域：

- [主工具欄](#id2051EA0G05Z)
- [輔助工具欄](#id2051EA0J0Y4)
- [左面板](#id2051EA0M0HS)
- [內容編輯區域](#id2051EB000UI)
- [右面板](#id2051EB003YK)

下面的小節詳細介紹了Web編輯器的各個部分。

## 主工具欄 {#id2051EA0G05Z}

主工具欄位於Web編輯器介面的頂部，它提供了Web編輯器中提供的檔案級功能和各種創作模式。 頂部工具欄中的可用功能說明如下：

**儲存全部** - ![](images/SaveFloppy_icon.svg)

保存在所有開啟的主題中所做的更改。 如果在Web編輯器中開啟了多個主題，請按一下 **全部保存** 或使用 **克特爾**+**S** 快捷鍵將所有文檔保存在一次按一下中。 您不必單獨保存每個文檔。

>[!NOTE]
>
> 「保存」操作不會建立新版本的主題。 要建立新版本，請選擇另存為新版本。

**另存為新版本** - ![](images/save-revision-icon.png)

保存您在主題中所做的更改，並建立新的主題版本。 如果您正在處理新建立的主題，則版本資訊顯示為 **無**。

![](images/save-all-first-version-none_cs.png){width="800" align="left"}

版本號隨為主題或映射檔案建立的每個新版本而更改。

當您選擇使用 **另存為新版本**，將顯示以下對話框：

![](images/save-as-new-version-dialog.PNG){width="300" align="left"}

輸入注釋和版本標籤以標識更改，然後按一下 **保存** 建立檔案的新版本。

選擇 *另存為新版本*，該主題的第一個版本是在DAM中建立的，該版本也成為您主題的當前活動版本。 稍後，如果恢復到主題的較舊版本，則該版本將成為主題的當前活動版本。

如果您的管理員已預配置了版本標籤，則您將在下拉清單中看到這些標籤。 可以從可用標籤清單中選擇標籤並保存文檔。

![](images/web-editor-pre-defined-labels.PNG){width="300" align="left"}

在保存主題時，可以添加一個注釋，指定您在主題中所做的更改。 此注釋顯示在主題的版本歷史記錄中。

如果您的主題正在審閱中，您的審閱者將收到通知，說明該主題的較新版本可用。 他們可以輕鬆訪問文檔的最新版本並繼續查看您主題的最新版本。

將指針懸停在主題標題上時，將顯示檔案路徑和版本號。

![](images/mouse-hover-on-title_cs.png){width="800" align="left"}

>[!NOTE]
>
> 一旦您的主題版本可用，您也可以將標籤添加到您的主題中。 然後，這些標籤可用於建立用於發佈特定版本的文檔的基線。 有關在主題中使用標籤的詳細資訊，請參閱 [使用標籤](web-editor-use-label.md#)。

**撤消和重做** - ![](images/Undo_icon.svg) / ![](images/Redo_icon.svg)

撤消或重做上一個操作。

**刪除元素** - ![](images/Delete_icon.svg)

刪除當前選定的元素或放置游標的元素。

**查找和替換** - ![](images/FindAndReplace_icon.svg)

「查找和替換」(Find and Replace)功能在「作者」(Author)和「源」(Source)視圖模式下可用。 「查找和替換」文本欄出現在主題編輯區域的底部。 可以使用快捷鍵 **CTRL鍵**+**F** 調用「查找」和「替換」欄。

![](images/find-replace-bar.png){width="800" align="left"}

使用設定表徵圖\(![](images/settings-find-replace-icon.svg)\)，可以切換 **忽略大小寫** 和 **僅全字** 搜索選項。 要執行不區分大小寫的搜索，請開啟\（或選擇\） **忽略大小寫** 的雙曲餘切值。 否則，如果要執行區分大小寫的搜索，請關閉\（或取消選擇\） **忽略大小寫** 的雙曲餘切值。 您也可以選擇搜索整個單詞。

搜索是即時的，這意味著在您鍵入搜索片語或單詞時 **查找** 欄位中，將立即搜索並選擇該術語。 同樣，要替換主題中的文本，請在相應欄位中輸入搜索詞及其替換，然後按一下 **替換** 或 **全部替換** 按鈕

在「源」視圖中，「查找和替換」對於搜索特定元素或屬性非常有用。 例如，如果要替換 `@product` 屬性，可以從「源」視圖輕鬆完成。 「作者」視圖不允許您基於屬性或元素進行搜索。 但是，在使用 **全部替換** 功能，因為它可能會覆蓋XML代碼。

**編輯器設定** - ![](images/editor_settings_icon.svg)

編輯器設定僅對管理用戶可用。 使用首選項，管理員可以配置以下設定：

>[!NOTE]
>
> 如果要更新任何預設設定，則應重新開啟文檔以使更改生效。

- **常規**:「常規」設定允許您將字典配置為與Web編輯器一起使用。 此頁籤包含三個部分： **拼寫檢查**。 **條件**, **創作**。

   ![](images/editor-setting-general.png){width="650" align="left"}

   - **拼寫檢查**:有兩種選擇： **拼AEM寫檢查** 和 **瀏覽器拼寫檢查**。 預設情況下，編輯器使用「瀏覽器拼寫檢查」功能，其中使用瀏覽器的內部生成字典執行拼寫檢查。 您可以切換到「AEM拼寫檢查」來使AEM用字典，也可以自定義詞典以添加自定義單詞清單。 有關自定義詞典的AEM詳細資訊，請參見 *自定義AEM預設詞典* 安裝和配置Adobe Experience Manager指南as a Cloud Service。


   - **條件**

      - **在「作者」視圖中突出顯示條件文本**:選擇此項以在作者視圖中突出顯示條件文本。 使用為條件定義的顏色加亮條件內容。

      - **使用條件屬性驗證**:選擇此項以允許驗證為屬性定義的值。 這會阻止您添加任何不正確的值。

      - **在主題方案面板中顯示帶標題的密鑰**:選擇此選項可在主題方案中顯示鍵和標題。 如果未選擇此選項，則只顯示標題。 例如，此處還顯示鍵「os」、「avience」和「other」以及標題。

         ![](images/subject-scheme-title.png){width="550" align="left"}

      - **在條件面板中顯示主題方案**:選擇此選項可在條件面板中查看主題方案。 如果取消選擇此選項，則定義的條件將顯示在條件面板中。
   - **編寫**

      - **啟用全部替換**:選擇此選項可查看「查找和替換」面板中的「全部替換」表徵圖。


**面板**:此設定控制編輯器左面板中顯示的面板。 可以切換開關以顯示或隱藏所需面板。

![](images/editor-setting-panel.png){width="650" align="left"}

>[!NOTE]
>
> 如果已配置了自定義面板，則它也會出現在面板清單中。 可以切換開關以顯示或隱藏自定義面板。 有關配置的詳細資訊，請參閱 *在左面板中配置自定義面板* 安裝和配置Adobe Experience Manager指南as a Cloud Service。

- **元素清單**:作為管理員，您可以控製作者可以使用 [插入元素](#id204SG30105Z) 的子菜單。 「元素清單」設定允許您根據DITA規範指定元素的名稱，並指定要使用的標籤，而不是DITA定義的元素名稱：

   ![](images/editor-setting-element-list.png){width="650" align="left"}

在上面的螢幕截圖中， `b` 元素的標籤為Bold , `codeblock` 為「代碼塊」加上其他元素。 如果選擇 **僅使用上面的元素** 選項，則此清單中的有效元素\（當前插入點\）將顯示在「插入元素」彈出窗口中。

在以下螢幕快照中，在當前上下文中僅顯示前一個螢幕快照中4個配置元素中的3個：

![](images/editor-setting-insert-element-list.PNG){width="300" align="left"}

- **屬性清單**:與「元素清單」(Elements List)類似，您可以控制要在元素的屬性清單中顯示的屬性清單及其顯示名稱。 在以下螢幕快照中，僅將3個屬性配置為顯示在元素的屬性清單中：

![](images/editor-setting-attributes-list.png){width="650" align="left"}

使用此設定時，當您嘗試向元素添加屬性時，您只能看到清單中配置的屬性清單。

![](images/editor-setting-add-attributes-list.png-to-element.PNG){width="300" align="left"}

- **顯示屬性**:與「屬性清單」(Attributes List)一樣，您可以控制要顯示在元素屬性清單中的屬性清單。 預設情況下，四 **顯示屬性**  — 受眾、平台、產品和道具已配置為顯示在元素的屬性清單中。 您還可以使用 **添加** 表徵圖。 您還可以使用 **刪除** 表徵圖

為元素定義的屬性顯示在「佈局」和「大綱」視圖中。

   - **將源版本標籤傳播到目標版本**:選擇此選項可將源檔案版本的標籤傳遞到已轉換的檔案。 預設情況下，此選項為禁用狀態。

   ![](images/editor-setting-translation.png){width="550" align="left"}


**使用者偏好設定** - ![](images/user_preference_editor_icon.svg)

「用戶首選項」(User Preferences)可供所有作者使用。 使用首選項，作者可以配置以下設定：

![](images/user_preference_editor.PNG){width="550" align="left"}

- **主題**:可以從編輯器的「光」、「光」、「暗」或「最暗」主題中進行選擇。 在「最淺」主題中，工具欄和面板使用更淺的灰色背景。 對於「淺」主題，工具欄和面板使用淺灰色背景。 對於「最暗」主題，工具欄和面板使用更深的黑色背景。 對於「深色」主題，工具欄和面板使用黑色背景。 在所有主題中，內容編輯區域以白色背景顯示。

- **資料夾配置檔案**:「資料夾配置檔案」控制與條件屬性、創作模板、輸出預設和Web編輯器配置相關的各種配置。 預設情況下顯示全局配置檔案。 此外，如果管理員已在系統中配置了資料夾配置檔案，則這些資料夾配置檔案也會顯示在「資料夾配置檔案」清單中。

   管理員可以在資料夾配置檔案中定義的Web編輯器配置包括：自定義用戶介面，包括工具欄表徵圖、Web Editor的佈局、代碼段和根映射。 有關詳細資訊，請參閱 *配置全局或資料夾級配置檔案* 安裝和配置Adobe Experience Manager指南as a Cloud Service。

   >[!NOTE]
   >
   > 當前資料夾配置檔案的名稱將作為主工具欄中「用戶首選項」表徵圖的標籤顯示。

- **基本路徑**:預設情況下，從Web編輯AEM器訪問儲存庫時，會從/content/dam位置顯示資產。 您的工作資料夾很可能是/content/dam/資料夾內的幾個資料夾。 每次要點擊幾下才能到達工作資料夾。 您可以設定工作資料夾的基本路徑，然後「儲存庫視圖」會提前顯示該位置的內容。 這減少了訪問工作資料夾的時間。 此外，當您在主題中插入任何引用或媒體檔案時，檔案瀏覽位置將從基本路徑中設定的資料夾開始。

- **選擇根映射**:選擇DITA映射檔案以解析鍵引用或術語表項。 所選根映射的優先順序最高，用於解析鍵引用。 有關詳細資訊，請參閱 [解析鍵引用](map-editor-other-features.md#id176GD01H05Z)。


>[!NOTE]
>
> 如果不想使用任何根映射，請確保 **選擇根映射** 欄位為空。

## 輔助工具欄 {#id2051EA0J0Y4}

在Web編輯器中開啟主題進行編輯時，將顯示輔助工具欄。 輔助工具欄中的可用功能說明如下：

**插入元素** - ![](images/Add_icon.svg)

在當前或下一個有效位置插入有效元素。 如果您在塊元素中工作，如 `note`，然後使用「插入元素」表徵圖在 `note` 的子菜單。 在下面的螢幕快照中，在p \(paragraph\)元素中插入了一個注釋元素：

![](images/note-in-para-insert-element_cs.png){width="800" align="left"}

如果在注釋元素中按Enter鍵，則會在注釋元素本身中建立新段落。 要在注釋外插入新元素，請在元素breadcrumb中按一下p元素\（在螢幕截圖\中突出顯示），然後按一下「插入元素」表徵圖或按 ***Alt鍵***+***輸入*** 開啟「插入元素」彈出窗口。 然後，選擇所需元素，然後按Enter鍵，在註解元素後插入選定元素。

當出現閃爍的塊游標時，還可以在兩個元素之間添加元素。

![](images/Block-cursor.png){width="300" align="left"}

例如，如果您正在處理DITA主題，並且塊游標在簡短描述和主體之間閃爍，則可以添加 `prolog` 元素，然後添加版權、作者和其他詳細資訊。

輸入新元素的另一種方法是使用上下文菜單。 按一下右鍵文檔中的任何位置以調用上下文菜單。 從此菜單中選擇「插入元素」以顯示「插入元素」對話框，然後選擇要插入的元素。

![](images/insert-element-before-after.png){width="300" align="left"}

**插入段落** - ![](images/Paragraph_icon.svg)

在當前或下一個有效位置插入段落元素。

**插入/刪除編號清單** - ![](images/TextNumbered_icon.svg)

在當前或下一個有效位置建立編號清單。 如果您在編號清單上並按一下此表徵圖，則項目將轉換為普通段落。

**插入/刪除項目符號清單** - ![](images/BulletList_icon.svg)

在當前或下一個有效位置建立項目符號清單。 如果您位於項目符號清單中，並按一下此表徵圖，則項目將轉換為普通段落。

**插入表格** - ![](images/Table_icon.svg)

在當前或下一個有效位置插入表。 按一下「插入表」表徵圖以開啟「插入表」對話框：

![](images/table-properties.png){width="550" align="left"}

可以指定表中所需的行數和列數。 如果要將第一行保留為表標題，請選擇「將第一行設定為標題」選項。 要將標題添加到表中，請在「標題」欄位中輸入該標題。

插入表後，可以使用上下文菜單修改表。

![](images/table-context-menu_cs.png){width="550" align="left"}

使用表的上下文菜單，可以：

- 插入單元格、行或列

- 在右和下方向合併單元格

- 水準或垂直拆分單元格

- 刪除單元格、行或列

- 從表中建立代碼段

- 生成ID


還可以定義表的多個單元格、整行或列的屬性。 例如，要對齊表格單元格，請拖動並選擇所需的單元格。 在「內容屬性」面板\（右側）中， **類型** 更改 **多條目**。 在「其他屬性」部分，選擇 `@valign` 屬性下拉清單中的屬性。 從值下拉清單中，選擇要應用於選定表格單元格的所需文本對齊方式。

![](images/align-table-cell_cs.png){width="800" align="left"}

**插入影像** - ![](images/Image_icon.svg)

在當前或下一個有效位置插入影像。 按一下「插入影像」表徵圖以開啟「插入影像」對話框，然後搜索並選擇要插入的影像。

>[!NOTE]
>
> 還可以通過將影像從本地系統拖放到文章中來添加影像。 在這種情況下，使用 **上載資產** 工作流。  有關詳細資訊，請參閱 **上載資產** 工作流 [左面板](web-editor-features.md#id2051EA0M0HS) 的子菜單。


![](images/insert-image.png){width="650" align="left"}

可以在「插入影像」對話框中為影像添加影像/表徵圖標題和替代文本。

通過在頂部的「要搜索的類型」欄中輸入檔案名，您可以搜索所需的影像檔案，還可以按路徑\（要在\中搜索）、集合、檔案類型和標籤篩選搜索結果。 找到所需的影像檔案後，選擇該檔案，然後按一下「選擇」將影像插入文檔。 可以插入各種格式的影像檔案，如 `.png`。 `.svg`。 `.gif`。 `.jpg`。 `.eps`。 `.ai`。 `.psd`。

插入影像後，可以從「內容屬性」面板更改高度、寬度、位置和屬性。 按一下影像檔案，然後在右欄的「內容屬性」面板中進行更改。

![](images/image-properties.png){width="800" align="left"}

「源」欄位顯示插入的映像檔案的UUID。 通過將滑鼠指針懸停在「源」欄位上，可以找到插入影像檔案的完整路徑。 路徑顯示在工具提示中。

可以通過為影像檔案提供「高度」或「寬度」值來調整影像的大小。 自動地保持影像的縱橫比。 如果需要，還可以通過按一下鎖定表徵圖\（「維護長寬比」的表徵圖\）並提供「高度和寬度」值來選擇不維護影像檔案的長寬比。

還可以將影像的「放置」設定指定為「內聯」或「中斷」。 如果選擇使用「分隔」放置選項，則可以選擇對齊影像\（左、中或右\）的位置。

也可以通過在 **屬性** 的子菜單。

>[!NOTE]
>

**合併** - ![](images/merge-icon.svg)

在多作者環境中工作時，很難跟蹤其他作者在主題或地圖中所做的更改。 「合併」功能不僅允許您查看更改，還允許您在文檔的最新版本中保留哪些更改。

**合併主題檔案**

要合併主題中的更改，請執行以下步驟：

1. 在Web編輯器中開啟主題。

1. 按一下 **合併**。

   將出現「合併」對話框。

   ![](images/merge-changes-in-topic.png){width="550" align="left"}

1. *\（可選\）* 您還可以從儲存庫中的其他位置瀏覽並選擇新檔案。

1. 選擇要與當前版本的檔案進行比較的檔案版本。

1. 從「選項」中選擇：

   - **跟蹤所選版本的更改**:此選項以跟蹤更改的形式顯示所有內容更新。 然後，您可以選擇一次接受或拒絕文檔中的更改，或者一次接受更改。

   - **還原為所選版本**:此選項將文檔的當前版本還原為所選版本。 此選項不會授予您對接受或拒絕的內容的任何控制。

1. 按一下 **完成**。

1. 如果選擇了 **跟蹤從選定版本更改的** 選項，則選定版本的所有更改將顯示在右側面板的「跟蹤更改」功能中。

   您可以選擇接受或拒絕「跟蹤的更改」面板中的所有注釋，或接受或拒絕單個注釋。


**合併映射檔案**

要合併映射檔案中的更改，請執行以下步驟：

1. 在Web編輯器中開啟映射。

1. 按一下 **合併**。

   將出現「合併」對話框。

   ![](images/merge-changes-in-map.png){width="550" align="left"}

1. *\（可選\）* 您還可以從儲存庫中的其他位置瀏覽並選擇新檔案。

1. 選擇要與當前版本的檔案進行比較的檔案版本。

1. 從「選項」中選擇：

   - **跟蹤所選版本的更改**:此選項以跟蹤更改的形式顯示所有內容更新。 然後，您可以選擇一次接受或拒絕文檔中的更改，或者一次接受更改。

   - **還原為所選版本**:此選項將文檔的當前版本還原為所選版本。 此選項不會授予您對接受或拒絕的內容的任何控制。

1. 按一下 **完成**。

   1. 如果選擇了 **跟蹤從選定版本更改的** 選項，則所選版本的所有更改將顯示在跟蹤的更改面板\（右側）中。

      您可以選擇接受或拒絕「跟蹤的更改」面板中的所有更改，或接受或拒絕映射檔案中的單個更改。


**版本歷史記錄** - ![](images/version-history-web-editor-ico.svg)

指AEM南提供了查看為主題檔案建立的版本的各種方法，也提供了恢復為特定版本的方法。 但是，這些功能大多在Web編輯器之外可用。

Web編輯器中的「版本歷史記錄」功能不僅允許您檢查活動主題上的可用版本和標籤，還允許您靈活地從編輯器本身恢復到任何版本。

要訪問版本歷史記錄並還原到主題的特定版本，請執行以下步驟：

1. 在Web編輯器中開啟主題。

1. 按一下 **版本歷史記錄**。

   將顯示「版本歷史記錄」對話框。

   ![](images/version-history-dialog-web-editor.png){width="550" align="left"}

1. 選擇要還原到的主題版本 **選擇版本** 的子菜單。

   >[!NOTE]
   >
   > 如果某個版本已應用標籤，則它們還會與版本號一起顯示\（括弧\中）。

   從下拉清單中選擇一個版本後，「還原到選定版本」選項將可用。 預覽窗口顯示當前版本與主題選定版本之間的差異。

   ![](images/version-history-revert-diff-dialog-web-editor.png){width="550" align="left"}

1. 按一下 **還原為所選版本** 將工作副本還原為選定的主題版本。

   將出現「還原版本」對話框。

   ![](images/version-history-revert-dialog-save-working-copy.png){width="550" align="left"}

1. \(*可選*\)提供還原到早期版本的原因。 您還可以建立主題的當前活動工作副本的新版本。

1. 按一下 **確認。**

   檔案的工作副本將還原為所選版本。 如果選擇建立當前活動工作副本的新版本，則還會使用所有工作更改建立檔案的新版本。


恢復到早期版本時，會顯示一個可視提示，指示您當前正在處理的版本不是最新版本。

![](images/older-version-visual-cue.png){width="800" align="left"}

**版本標籤管理** -  ![](images/version-label-icon.svg)

標籤可幫助您確定特定主題在DDLC \（文檔開發生命週期\）中所處的階段。 例如，在處理主題時，可以將標籤設定為「已批准」。 一旦發佈了一個主題並讓客戶可用，您就可以將「已發佈」標籤分配給該主題。

參AEM考線允許您以自由格式文本格式指定標籤或使用一組預定義標籤。 自定義標籤將允許系統中的任何作者根據其選擇指定標籤。 這給了靈活性；但是，它在系統中引入了不一致的標籤。 為克服此問題，管理員可以配置一組預定義的標籤。 有關配置預定義標籤的詳細資訊，請參見 *配置和自定義XML Web編輯器* 安裝和配置Adobe Experience Manager指南as a Cloud Service。

這些標籤以下拉清單的形式顯示給作者，無論他們需要指定標籤。 這確保系統中只使用預定義的一致標籤。

您可以通過不同的方法將標籤應用於主題 —  [版本歷史記錄](web-editor-use-label.md#) UI中的面板， [基線](/help/tutorials/user-guide/generate-output-use-baseline-for-publishing.md#id184KD0T305Z) UI和Web編輯器。 Web編輯器中的「版本標籤」功能為作者提供了將標籤分配給其主題的快速而簡便的方法。

要從Web編輯器向主題添加標籤，請執行以下步驟：

1. 在Web編輯器中開啟主題。

1. 按一下 **版本標籤**。

   此時將顯示「版本標籤管理」對話框。

   ![](images/version-label-management-dialog.png){width="650" align="left"}

   「版本標籤管理」對話框分為兩部分 — 左面板有一個可用於主題的版本清單，以及標籤下拉清單\（或一個用於輸入標籤\的文本框），右面板帶有主題的預覽。

1. 選擇要應用標籤的版本。

   從版本清單中選擇主題的其他版本時，預覽面板將顯示當前版本與主題選定版本之間的更改

   >[!NOTE]
   >
   > 如果已在某個版本上應用了標籤，則標籤將顯示在下拉清單中版本號的旁邊以及「選擇版本」清單的下方。 通過按一下\(**x**\)標籤旁的表徵圖。

1. 如果管理員已定義了標籤清單，則會顯示標籤的下拉清單，您可以從中選擇要應用的標籤。 可以從下拉清單中選擇多個標籤。

   否則，將顯示一個文本框，您可以在其中輸入要添加到主題的標籤。

   >[!NOTE]
   >
   > 不能對主題的多個版本應用同一標籤。 如果嘗試關聯現有標籤，則系統會為您提供一個選項，可將其從現有版本中刪除，並將其應用於主題的選定版本。

1. 按一下 **添加標籤**。

1. 在「應用標籤」確認消息中，選擇 **移動標籤** 選項將標籤從現有版本移動到選定版本。 如果未選擇此選項，並且有標籤被分配給主題的其他版本，則不會將標籤移動到所選主題的版本。 在標籤應用程式過程中忽略這些標籤。


**建立審閱任務** -  ![](images/create-review-task-icon.svg)

1. 選擇術語表面板以查看根映射中存在的術語表術語。 您可以拖放這些術語，以將它們添加到開啟的主題中。

- **辭彙表鍵**:從DITA映射中選擇要用於選定主題中搜索的辭彙表鍵。 所選鍵將顯示在下面。 通過按一下 **刪除** 表徵圖

- **主題**:選擇 **當前主題** 在Web編輯器中開啟，所有 **開啟的主題** 或 **當前映射** 在映射編輯器中編輯以搜索術語。
- **按狀態篩選主題**:您可以選擇將搜索限制為具有選定文檔狀態的主題。 這些主題可以處於「草稿」、「編輯」、「正在審閱」、「已批准」、「已審閱」、「完成」狀態，或處於組織配置的任何一種狀態。
- **操作**:可以選擇搜索辭彙表鍵 **手動處理每個主題** 或 **自動處理所有主題**。 如果您選擇 **手動處理每個主題**，它將提示您在轉換每個主題中的每個術語之前進行確認。 如果您選擇 **自動處理所有主題**，它自動轉換所有主題中的所有術語。
- **轉換**:可以轉換已搜索的 **文本到術語表術語** 或 **術語表術語到文本。**
- **選項**:可以從以下選項中選擇：
   - **區分大小寫匹配**:搜索詞以查找具有相同外殼的匹配項。 例如，「USB」與「usb」不匹配。
   - **僅轉換第一個實例**:如果搜索項的多個實例存在於主題中，則只轉換第一實例。
   - **轉換前簽出檔案**:在轉換術語之前，檢出搜索的檔案。
   - **轉換後建立新版本**:在術語轉換完成後建立主題的新版本。
- **下一個** 按鈕 **為每個主題手動** 的雙曲餘切值。 按一下 **下一個** 以根據所選設定轉換每個主題的術語。 它提示在每個主題中轉換術語，並移到下一個檔案。 您可以選擇轉換術語或跳過術語並移到下一個術語。

   ![](images/manual-convert-skip.png){width="300" align="left"}

- **轉換** 按鈕 **自動處理所有主題** 的雙曲餘切值。 選擇 **轉換** 將文檔中找到的所有術語轉換為連結的辭彙縮寫。

清單 **已更新主題** 用轉換後的術語和 **帶錯誤的主題** 的上界。 懸停在\( ![](images/info-icon.svg)\)主題附近的表徵圖，查看錯誤的詳細資訊。

**主題方案** -  ![](images/subject_scheme_panel-icon.svg)

主題方案映射是DITA映射的一種專門形式，用於定義分類主題和受控值。 根據您的要求，您可以建立主題方案映射並在根映射檔案中引用它。 AEM參考線允許您定義主題方案中主題定義的嵌套層次結構。

您可以輕鬆地在主題方案映射中建立和使用主題方案。 將此映射添加為根映射後，主題方案將顯示在「主題方案」面板中。 「主題方案」面板以嵌套或分層方式顯示可用的主題方案。

輔助AEM線還支援嵌套級別主題方案映射，並且可以在根主題方案映射下定義多個主題方案。

以下示例說明如何在「參考線」中使AEM用主題方案。

1. 在所選工具中建立主題方案檔案。 以下XML代碼建立綁定XML的值的主題方案 `platform` 屬性。

   ```XML
   <?xml version="1.0" encoding="UTF-8"?>
   <!DOCTYPE subjectScheme PUBLIC "-//OASIS//DTD DITA Subject Scheme Map//EN" "subjectScheme.dtd">
   <subjectScheme id="GUID-4f942f63-9a20-4355-999f-eab7c6273270">
       <title>rw</title>
       <!-- Define new OS values that are merged with those in the unixOS scheme -->
       <subjectdef keys="os">
           <subjectdef keys="linux">    </subjectdef>
           <subjectdef keys="mswin">    </subjectdef>
           <subjectdef keys="zos">    </subjectdef>
       </subjectdef>
       <!-- Define application values -->
       <subjectdef keys="app" navtitle="Applications">
           <subjectdef keys="apacheserv">    </subjectdef>
           <subjectdef keys="mysql">    </subjectdef>
       </subjectdef>
       <!-- Define an enumeration of the platform attribute, equal to       each value in the OS subject. This makes the following values       valid for the platform attribute: linux, mswin, zos -->
       <enumerationdef>
           <attributedef name="platform">    </attributedef>
           <subjectdef keyref="os">    </subjectdef>
       </enumerationdef>
       <!-- Define an enumeration of the otherprops attribute, equal to       each value in the application subjects.       This makes the following values valid for the otherprops attribute:       apacheserv, mysql -->
       <enumerationdef>
           <attributedef name="otherprops">    </attributedef>
           <subjectdef keyref="app">    </subjectdef>
       </enumerationdef>
   </subjectScheme>
   ```

   ![](images/subject-scheme-panel.png){width="300" align="left"}

1. 以.ditamap副檔名保存檔案，並將其上載到DAM中的任何資料夾。

   >[!NOTE]
   >
   > 可以在父DITA映射中向主題方案檔案添加引用。

   ![](images/subject-scheme-root-map.png){width="550" align="left"}

1. 將父映射設定為 **用戶首選項**。 將此映射添加為根映射後，主題方案將顯示在「主題方案」面板中。

   ![](images/subject-scheme-user-preferences.png){width="400" align="left"}

1. 在Web編輯器中，開啟要使用主題方案定義的檔案。
1. 只需將所需主題方案拖放到您的內容上，即可將主題方案應用於您的內容。 然後，該內容以定義的顏色加亮。

   > 啟用 **全部替換** 表徵圖，系統管理員必須選擇 **啟用全部替換** 下 **常規** 頁籤 **編輯器設定**。


在整個系統中，一次只能執行一個替換所有操作，在執行該操作之前，您將看到「替換所有正在進行的」狀態。 您也可以中止中的替換所有操作或查看日誌報告。 如果中止該操作，您將在「收件箱」中收到有關該操作的通知。 替換選定檔案中的所有事件後，將顯示成功通知。

![](images/replace-all-in-progress.png){width="400" align="left"}

您還可以使用 **在映射中查找** 的 **選項** 的子菜單。 此選項顯示在儲存庫面板或映射視圖中開啟的映射。

![](images/map-options-menu.png){width="550" align="left"}

## 內容編輯區域 {#id2051EB000UI}

內容編輯區域是顯示主題或映射內容的位置。 您可以編輯此區域中的所有內容。 它提供您正在編輯的內容的WYSIWYG視圖。 您可以同時開啟多個主題，這些主題顯示在各自的頁籤中。 在檔案的頁籤下，元素的breadcrumb位於當前游標位置。 在內容編輯區域的右上角，將顯示當前主題的版本號。

![](images/content-editing-area.png){width="650" align="left"}

## 右面板 {#id2051EB003YK}

右面板是一個永久面板，包含有關當前選定文檔的資訊。

>[!NOTE]
>
> 右面板可調整大小。 要調整面板大小，請將游標置於面板邊界上，游標將變為雙向箭頭，按一下並拖動以調整面板寬度。

右面板允許您訪問以下功能：

**內容屬性** -  ![](images/content-properties-icon.svg)

通過按一下右面板中的「內容屬性」表徵圖，可以訪問「內容屬性」功能。 「內容屬性」面板包含有關文檔中當前選定元素的類型及其屬性的資訊。 也可以通過從下拉清單中選擇屬性並指定屬性的值來添加屬性。

>[!NOTE]
>
> 即使您的主題包含引用的內容，也可以使用「屬性」面板在其上添加屬性。

如果管理員已為屬性建立了配置檔案，則您將獲得這些屬性及其配置的值。 使用內容屬性面板，您可以選擇這些屬性並將其分配給主題中的相關內容。 這樣，您還可以建立條件內容，然後可以使用這些內容建立條件輸出。 有關使用條件預設生成輸出的詳細資訊，請參見 [使用條件預設](generate-output-use-condition-presets.md#)。

![](images/properties-tab-attributes_cs.png){width="300" align="left"}

**檔案屬性** -  ![](images/topic-properties-icon.svg)

按一下右面板中的「檔案屬性」表徵圖，查看選定檔案的屬性。 「檔案屬性」(File Properties)包含以下兩部分：

**一般**

「常規」(General)部分允許您訪問以下功能：

![](images/file-properties-general.png){width="300" align="left"}

- **名稱**:顯示所選主題的檔案名。 檔案名超連結到選定檔案的屬性頁。
- **ID**:顯示所選主題的ID。
- **元資料標籤**:這些是主題的元資料標籤。 它們是從屬性頁的標籤欄位中設定的。
- **語言**:顯示主題的語言。 它是從屬性頁面的語言欄位中設定的。
- **建立時間**:顯示建立主題的日期和時間。
- **簽出者**:顯示簽出主題的用戶。
- **文檔狀態**:您可以選擇並更新當前開啟的主題的文檔狀態。 有關詳細資訊，請參閱 [文檔狀態&#x200B;](web-editor-document-states.md#)*。*

**注：** 可以將「檔案」屬性中各個欄位的屬性值複製到剪貼簿。

**引用**

「參照」(References)部分允許您訪問以下特徵：

![](images/file-properties-references.png){width="300" align="left"}

- **用於**:「在引用中使用」(Used In references)列出引用或使用當前檔案的文檔。
- **傳出連結：** 「傳出連結」列出當前文檔中引用的文檔。

將滑鼠懸停在檔案的引用上，在工具提示中獲取檔案的檔案路徑和UUID。

**注：** 所有「Used In（在中）」和「Outgoin（傳出）」引用都超連結到文檔。 您可以輕鬆開啟和編輯連結的文檔。

除開啟檔案外，還可以使用 **選項** 的子菜單。 可以執行的某些操作包括編輯、預覽、複製UUID、複製路徑、添加到收藏夾、屬性和開啟映射儀表板。

**審閱** -  ![](images/review-icon.svg)

按一下「審閱」(Review)表徵圖可開啟審閱面板，您可以在其中為當前開啟的文檔建立審閱任務。

![](images/review-panel-before-opening.png){width="300" align="left"}

如果已建立多個「審閱」項目，則可以從下拉清單中選擇一個項目並訪問審閱注釋。

使用審閱面板，可以查看和發佈對主題上提供的評論的答復。 您可以逐個接受或拒絕注釋。

有關詳細資訊，請參見 [地址審閱注釋](review-address-review-comments.md#)。

**跟蹤的更改** -  ![](images/track-change-icon.svg)

使用右面板的「跟蹤的更改」功能，可以查看文檔中所有更新的資訊。 您還可以搜索對文檔進行的任何特定更新。

>[!NOTE]
>
> 「跟蹤的更改」功能顯示已使用主工具欄的「啟用/禁用跟蹤更改」功能跟蹤的所有更新。 有關詳細資訊，請參閱 [啟用/禁用跟蹤更改](#id205DF0203Y4)。
