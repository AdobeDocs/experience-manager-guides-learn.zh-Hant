# Web編輯器檢視 {#id204GK0D0V5Z}

AEM參考線的Web編輯器支援以三種不同的模式或檢視來檢視檔案：

## 作者

這是典型的Web編輯器「What You Get \(WYSISYG\)(獲取\(WYSISYG\))」視圖。 您可以像在任何一般RTF編輯器中一樣編輯主題。 在「作者」檢視中，您有選項可儲存檔案的修訂、尋找和取代內容、插入元素、插入超連結、插入內容參考等。

>[!NOTE]
>
> 使用內容參考時，參考的內容也會以藍色顯示在「作者」檢視中。 引用的內容不可編輯。

## 來源

「源」視圖顯示組成主題的基礎XML。 如果您習慣直接使用XML，則應使用「源」視圖。 除了在此視圖中進行常規文本編輯外，您還可以使用智慧目錄添加元素和屬性，或查找和替代文字、元素或屬性。

- 要調用智慧目錄，請將游標放在要插入新元素的任何元素標籤的末尾，然後輸入&quot;&lt;&quot;。 編輯器會顯示所有可在該位置插入的有效XML元素的清單。 使用箭頭鍵選擇要插入的元素，然後按Enter鍵。 當您輸入結尾括弧&quot;\>時，會自動新增元素的結尾標籤。

   ![](images/smart-catalog-elements.png){width="400" align="left"}

- 您也可以從「源」視圖輕鬆更改元素。 例如，如果您變更 `p` 元素 `note`，然後結尾 `p` 標籤會自動變更為 `/note`. 如果您以錯誤的元素取代元素，則會立即顯示「驗證錯誤」。

- 如果要向元素添加屬性，請將游標放在元素標籤內，然後按空格鍵。 該元素的有效屬性清單顯示在智慧型目錄中。 使用方向鍵選取所需元素，然後按Enter以插入元素。 要指定屬性的值，請輸入等號\(=\)，編輯器將自動輸入開頭和結尾引號&quot;&quot;，您可在其中指定屬性的值。

   ![](images/smart-catalog-attribute.png){width="350" align="left"}

- 在「原始碼」視圖中，有一個「自動縮進」選項，它以顯示且易於閱讀的格式重新組織XML代碼。 此外，如果您選取任何文字，並從「作者」切換至「來源」或從「來源」切換至「作者」檢視，選取的文字也會在另一個檢視中反白顯示。
- 「源」視圖中的另一個強大功能是文檔中的XML驗證。 如果開啟包含無效XML的文檔，則會在「源」視圖中開啟該文檔，其中包含有關無效XML的資訊。 例如，在下面的螢幕截圖中，在「分析錯誤」彈出窗口中提供了有關錯誤XML的確切資訊。

   ![](images/invalid-topic-xml.png){width="650" align="left"}

   在上述螢幕擷取中，交叉加亮用於指向包含錯誤XML的行。

- 「查找和替換」功能允許您在「源視圖」中搜索任何文本、元素或屬性。
如需詳細資訊，請參閱 **查找和替換** 功能說明，請參閱 [主工具欄](web-editor-features.md#id#id2051EA0G05Z) 區段。

- 「源視圖」提供了許多快捷方式，幫助您快速導航和處理文檔。 下表列出支援的動作及其快捷鍵：

## 預覽

在「預覽」模式中開啟主題時，會顯示當使用者在瀏覽器中檢視主題時，主題的顯示方式。 若是DITA地圖，則會顯示地圖的預覽，其中會顯示地圖中所有主題的單一複合檔案。

預覽模式提供下列功能：

- [根據條件篩選器檢視內容](#id2114BI00VXA)
- [檢視追蹤變更標籤](#id2114BJ00CE8)
- [將主題匯出為PDF](#id2114BL00B5U)

### 根據條件篩選器檢視內容 {#id2114BI00VXA}

如果您已在主題或對應中使用條件，這些條件會顯示在「篩選器」面板中。 依預設，會選取所有條件，並顯示整個內容。 如果您取消選取條件，則具有該條件的內容會從檢視中移除。 您也可以選擇反白標示有條件的內容。

下圖顯示一個使用兩個條件的主題： `Audience` 和 `Product`. 有條件的內容會以黃色背景強調顯示。

![](images/preview-filters.png){width="800" align="left"}

### 檢視追蹤變更標籤 {#id2114BJ00CE8}

如果文檔包含跟蹤更改標籤\（或視覺提示\），則您也可以使用這些標籤預覽文檔，或者不使用這些標籤。 預覽檔案時，右側面板包含「篩選」和「追蹤」選項。

![](images/preview-tracking_cs.png){width="400" align="left"}

有三個 **追蹤** 可以選擇的選項：

- **無標籤**:在此視圖中，所有插入和刪除都被接受，並呈現文檔的簡單視圖。 在此檢視中，您不會看到任何追蹤變更標籤。
- **原始**:在此視圖中，所有插入都被拒絕，所有刪除都被恢復，然後顯示預覽。 簡單地說，在啟用跟蹤更改模式之前，您將獲得文檔的原始形式。
- **顯示標注**:在此檢視中，您會收到插入和刪除內容的所有標籤。

   下圖顯示的是地圖檔案的Markup預覽：

   ![](images/preview-map-with-track-changes.PNG){width="800" align="left"}


### 將主題匯出為PDF {#id2114BL00B5U}

PDF是在文檔開發週期的每個可能階段使用的最常見的輸出格式之一。 AEM指南可讓您彈性產生個別主題或整個對應檔案的PDF。 「匯出為PDF」功能可讓作者、發佈者或管理員輕鬆產生個別主題的PDF輸出。 它使用儲存在資料夾層級設定檔的DITA-OT設定來產生PDF。

此功能支援下列功能：

- 生成主題當前活動工作副本的PDF。
- 接受DITA-OT轉換名稱和命令列引數以產生PDF。
- 在本地系統上保存生成的輸出。
- 在產生輸出之前，先解析主題中使用的索引鍵和內容參考。

若要將主題匯出為PDF，請執行下列步驟：

1. 在預覽模式中開啟主題。

1. 按一下 **匯出為PDF** \(![](images/export-as-pdf-icon.svg)\)圖示。

   將顯示「導出為PDF」對話框。

   ![](images/export-as-pdf-dialog.png){width="350" align="left"}

1. *\（可選\）* 指定DITA-OT轉換名稱以及您要使用的任何命令列引數。

1. 按一下&#x200B;**下載**。

   >[!NOTE]
   >
   > 請確定您已在瀏覽器設定中啟用快顯視窗，否則PDF將不會下載。

   PDF會在新索引標籤中產生並開啟，或者您會看到對話方塊，在本機系統上儲存PDF。
